# prd2proto 资深设计师评审标尺 (Senior Review Rubric)

> 用于评估 prd2proto v2 设计推理链路的产出是否接近资深设计师水准。
> 评分对象：真实 PRD 端到端跑出的 stage outputs。
> 评分者：资深设计师（人工）或基于本标尺的结构化自评。

---

## 评分刻度（每项 0-5 分）

| 分值 | 含义 |
|------|------|
| 0 | 缺失（该维度完全没有产出） |
| 1 | 形式存在（有字段，但内容空泛/套话） |
| 2 | 初级可用（junior 水平，能用但需大改） |
| 3 | 中级可用（mid 水平，方向对，需调整） |
| 4 | 资深可评审（senior 愿意在其上评审/微调，而非重做） |
| 5 | 资深可直接复用（senior 认可，可直接进入下一环节） |

**关键阈值**：≥4 分 = 达到"资深设计师可评审"的目标线（CLAUDE.md 上限标准）。

## 判定标准

- **总分 ≥48（平均4分）**：达到"资深设计师可评审"水准 ✅
- **总分 36-47（平均3分）**：中级可用，需针对性改进
- **总分 <36**：仍偏初级，需重点修 prompt/runtime
- **任一核心维度（D1/D2/D3/D9/D11/D12）<3**：该维度必须列入下轮修复

---

## 使用说明

1. 真实 PRD 端到端跑完后，对照每个 stage output 逐维度打分
2. 每项必须填"证据"（引用实际输出片段），禁止凭印象打分
3. 评分结果写入 `run-report.md`
4. <3 分的维度 → 进入 Batch 2 修复清单
