# 参考案例:有竞品矩阵支撑的品牌策略

> 本案例展示当 `competitor_matrix` 存在且数据充分时,如何产出基于竞品证据的品牌策略。

## 场景

用户输入产品简介与目标用户,upstream 子技能 `competitive-analysis` 已产出竞品矩阵。

## 输入

### product_brief

```
FlowForge 是一款面向技术创业者的项目管理工具,核心功能:
- 任务管理与看板
- 团队协作与文档
- 进度追踪与报告

目标市场:早期创业团队(3-15人)
价值主张:在功能之上提供情感支持,理解创业者的心理压力与不确定性
```

### target_user

```
- 25-40岁技术背景创业者
- 正在经历高不确定性与心理压力
- 已尝试多款项目管理工具,但感觉"冰冷"、"只关注任务完成"
- 需要工具之外的情感认同与支持
```

### competitor_matrix

```json
{

      "market_position": "企业级项目管理,覆盖大中小团队"
    },
    {
      "name": "Monday.com",

      "communication": "灵活性导向,强调'适应任何工作流''自定义'",

    },
    {
      "name": "Linear",

      "market_position": "技术团队专用,聚焦软件开发"
    }
  ],
  "status": "complete"
}
```

## Stage 1 产出:context_analysis

```markdown
## 产品核心价值

- 核心功能:任务管理/团队协作/进度追踪(与竞品功能对齐)
- 目标市场:早期创业团队(3-15人),聚焦技术创业者
- 价值主张:功能价值(项目管理)+ 情感价值(心理支持与认同)

## 目标用户洞察

- 人口统计:25-40岁技术背景创业者
- 核心痛点:不确定性/心理压力/孤独感,现有工具"冰冷"
- 情感需求:被理解/被支持/被认可,而非只是任务完成
- 关系类型:期待"伙伴"型关系(非纯工具)

## 竞品态势判定

**竞品共性(crowded zone):**
- 所有竞品都强调功能(效率/灵活/速度)
- 视觉风格偏理性(清爽/多彩/极简),情感共鸣弱
- 沟通方式聚焦任务完成,未触及用户心理层面

**差异化空间(white space):**
- 竞品未充分覆盖的情感支持维度
- 创业者心理旅程的陪伴与认同
- 从"冰冷工具"到"温暖伙伴"的定位转换

**差异化判定基础**: competitor_matrix

## 品牌战略机会点

### 情感化价值方向
用户痛点(心理压力/孤独感)+ 产品价值主张(情感支持)→ "让创业者感到被理解与陪伴"

### 定位候选方向
1. **情感支持型**:强调心理陪伴与认同
   - 优势:差异化明显,竞品未覆盖
   - 劣势:可能被认为"软",功能感弱
   - 适用场景:目标用户对情感需求敏感

2. **功能效率型**:对标 Asana/Linear
   - 优势:市场认知清晰
   - 劣势:与大厂正面竞争,差异化弱
   - 适用场景:只关注功能的理性用户

3. **极简美学型**:对标 Linear
   - 优势:吸引设计敏感用户
   - 劣势:情感价值不突出
   - 适用场景:视觉驱动的用户

### 差异化空间
基于竞品矩阵,竞品集中在功能/效率维度,情感支持维度是明确的 white space。

### 取舍建议
**推荐:情感支持型定位**
- 选择理由:竞品未覆盖,与产品价值主张一致,满足目标用户情感需求
- 放弃了什么:放弃与大厂的功能参数竞争,可能在纯功能导向用户中吸引力弱
```

## 关键决策点

1. **差异化判定基础**: `basis: "competitor_matrix"`
   - 因为 `competitor_matrix.status == "complete"` 且有 3 个竞品
   - 差异化声明引用了具体竞品证据(Asana/Monday/Linear 的沟通方式)

2. **北极星的情感化**:
   - ✅ "让创业者在不确定性中感到被理解与支持"(情感价值)
   - ❌ "提供高效的项目管理工具"(功能描述)

3. **人格关键词覆盖三维度**:
   - 温暖(情感维度)
   - 进取/可信赖(价值维度)
   - 直接(行为维度)

4. **定位取舍说明**:
   - 选择:情感支持型
   - 未选:功能效率型/极简美学型
   - 取舍:放弃功能竞争,获得差异化空间
