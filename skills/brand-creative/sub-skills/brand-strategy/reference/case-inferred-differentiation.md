# 参考案例:竞品矩阵缺失时的差异化推断

> 本案例展示当 `competitor_matrix` 缺失或数据不足时,如何标注 [inferred] 并保守产出品牌策略。

## 场景

用户直接调用 brand-strategy 子技能,未提供 competitor_matrix,或 upstream 的 competitive-analysis 产出 `status: "insufficient_data"`。

## 输入

### product_brief

```
EduFlow 是一款面向 K12 教师的备课助手,核心功能:
- AI 生成教案与课件
- 知识点图谱自动关联
- 学情分析与个性化建议

目标市场:公立学校一线教师
价值主张:减轻教师备课负担,让教师有更多时间关注学生
```

### target_user

```
- 30-50岁公立学校教师
- 教学任务重,备课时间不足
- 对 AI 工具有兴趣但不熟悉
- 希望工具真正理解教学场景,而非只是技术堆砌
```

### competitor_matrix

```json
{
  "competitors": [],
  "status": "insufficient_data",
  "reason": "目标细分市场(K12 教师备课)竞品信息有限,无法提供充分对比数据"
}
```

## Stage 1 产出:context_analysis

```markdown
## 产品核心价值

- 核心功能:AI 生成教案/知识点图谱/学情分析
- 目标市场:公立学校一线教师(K12)
- 价值主张:功能价值(提效)+ 情感价值(减负与关注学生)

## 目标用户洞察

- 人口统计:30-50岁公立学校教师
- 核心痛点:备课时间不足,教学任务重
- 情感需求:被理解(教学场景复杂性),被支持(减负而非增加学习成本)
- 关系类型:期待"助手"型关系(非技术玩具)

## 竞品态势判定

**竞品矩阵状态**: insufficient_data
**原因**: K12 教师备课细分市场竞品信息有限,无法提供充分对比数据

**基于产品特性的推断:**
- 教育科技产品常见路径:技术导向(强调 AI/算法/数据)
- 本产品特性:强调"理解教学场景"与"减负",可能与纯技术导向产品形成差异
- 但未经竞品对标验证,无法确认实际差异化空间

**差异化判定基础**: inferred

## 品牌战略机会点

### 情感化价值方向
用户痛点(备课负担重)+ 产品价值主张(减负并关注学生)→ "让教师重获教学的从容与热情"

### 定位候选方向
1. **教学场景理解型**:强调懂教学,非技术堆砌
   - 优势:与用户情感需求一致
   - 劣势:未经竞品验证,不确定是否真有差异
   - 适用场景:用户对"技术玩具"有抵触

2. **AI 技术领先型**:对标技术导向竞品
   - 优势:市场认知清晰(AI 赛道)
   - 劣势:可能与用户情感需求不匹配(教师不关心算法)
   - 适用场景:技术敏感用户

3. **教师减负型**:强调省时高效
   - 优势:直击痛点
   - 劣势:可能过于功能化,情感价值弱
   - 适用场景:纯功能需求用户

### 差异化空间
基于产品特性推断,可能与纯技术导向产品形成"教学场景理解"差异,但未经竞品对标验证。

### 取舍建议
**推荐:教学场景理解型定位**
- 选择理由:与产品价值主张一致,满足用户情感需求("被理解")
- 放弃了什么:放弃 AI 技术参数竞争,可能在技术驱动市场中吸引力弱
- **风险提示**:差异化未经竞品验证,需后续补充竞品调研
```

## Stage 2 产出:brand_brief

```json
{
  "north_star": "让教师重获教学的从容与热情",
  "positioning": "面向 K12 教师的场景理解型备课助手,在 AI 之上提供对教学复杂性的深度理解",
  "differentiation": {
    "statement": "[inferred] 基于产品的'理解教学场景'特性,推断可能与偏技术导向的教育科技产品形成差异(强调场景理解 vs 技术参数),但未经竞品对标验证,仅为初步推断",
    "basis": "inferred"
  },
  "core_values": [
    "尊重教学专业性",
    "减负不添乱",
    "持续学习"
  ],
  "personality_keywords": [
    "沉稳",
    "专业",
    "体贴",
    "可靠"
  ],
  "target_user": "30-50岁公立学校教师,教学任务重,需要真正理解教学场景的减负工具",
  "_meta": {
    "positioning_tradeoffs": {
      "chosen_direction": "教学场景理解型定位",
      "unchosen_directions": [
        "AI 技术领先型",
        "教师减负型(纯功能导向)"
      ],
      "tradeoff_reasoning": "选择场景理解型,放弃了 AI 技术参数竞争,但与产品价值主张一致,满足用户'被理解'的情感需求。风险:差异化未经竞品验证,需后续补充竞品调研。"
    },
    "disclaimer": "本策略基线需资深品牌经理评审后方可商用。差异化判定基于 inferred,未经竞品对标验证,不构成完整竞品战略分析。建议后续补充竞品调研以验证差异化假设。"
  }
}
```

## 关键决策点

1. **差异化判定基础**: `basis: "inferred"`
   - 因为 `competitor_matrix.status == "insufficient_data"`
   - 差异化声明显式标注 `[inferred]` 并说明"未经验证"

2. **差异化声明的保守性**:
   - ✅ "推断可能与...形成差异,但未经验证"(保守推断)
   - ❌ "我们是市场唯一的场景理解型产品"(绝对化,违反约束)

3. **北极星的情感化**:
   - ✅ "让教师重获教学的从容与热情"(情感价值)
   - ❌ "提供最高效的 AI 备课工具"(功能描述)

4. **人格关键词覆盖三维度**:
   - 沉稳/体贴(情感维度)
   - 专业/可靠(价值维度)
   - (行为维度可从"沉稳"与"体贴"延展为"不急躁"与"主动提示")

5. **免责声明强化**:
   - 明确说明差异化未经验证
   - 建议后续补充竞品调研

6. **定位取舍说明风险**:
   - 除了说明选择与放弃,还明确风险("差异化未经验证")

---

**适用场景**: 用户直接调用 brand-strategy,未提供 competitor_matrix;或 workflow 运行时 competitive-analysis 失败/数据不足。
**对比案例**: 参见 `case-with-competitor-matrix.md`(竞品矩阵存在场景)。

## 关键差异对比表

| 维度 | 有竞品矩阵 | 无竞品矩阵(本案例) |
|------|-----------|-------------------|
| `differentiation.basis` | `"competitor_matrix"` | `"inferred"` |
| 差异化声明 | 引用具体竞品证据 | 标注 `[inferred]` 并说明未验证 |
| 声明语气 | 肯定("竞品普遍...我们...") | 保守("推断可能...但未验证") |
| 免责声明 | 基础版 | 强化版(建议补充调研) |
| 风险提示 | 无 | 明确风险(差异化未验证) |
