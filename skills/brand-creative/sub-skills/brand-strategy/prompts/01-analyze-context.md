# Stage 1: 分析上下文

你是资深品牌战略顾问。任务:分析产品定位/目标用户/竞品态势,识别品牌战略机会点。

## 输入

**product_brief**:
```
{{product_brief}}
```

**target_user**:
```
{{target_user}}
```

**competitor_matrix** (可选):
```json
{{competitor_matrix}}
```

**brand_audit_report** (可选):
```json
{{brand_audit_report}}
```

## 分析任务

### 1. 产品核心价值识别

- 核心功能与差异点
- 目标市场与细分人群
- 价值主张(功能价值 + 情感价值)

### 2. 目标用户洞察

- 人口统计特征
- 核心痛点与期待
- 情感需求(被理解/被支持/被认可/...)
- 用户与产品的关系类型(工具/伙伴/导师/...)

### 3. 竞品态势判定

**若 competitor_matrix 存在且 status="complete":**
- 分析至少 3 个竞品的视觉风格/沟通方式/市场定位
- 识别竞品集中的共性(crowded zone)
- 识别未被充分覆盖的差异化空间(white space)
- 记录:差异化判定基础 = "competitor_matrix"

**若 competitor_matrix 缺失或 status="insufficient_data":**
- 基于产品特性推断可能的差异点
- 明确标注:差异化判定基础 = "inferred"
- 说明:未经竞品对标验证,仅为初步推断

### 4. 品牌战略机会点

基于以上分析,提出:
- **情感化价值方向**: 用户内在需求与产品价值的交集
- **定位候选方向**: 至少 3 个可能的定位方向(功能导向/情感导向/权威导向/...)
- **差异化空间**: 基于竞品态势或产品特性的差异化机会
- **取舍建议**: 每个定位方向的优劣与适用场景

## 输出格式

以 Markdown 输出,包含以下部分:

```markdown
## 产品核心价值
[核心功能/目标市场/价值主张]

## 目标用户洞察
[人口统计/痛点/情感需求/关系类型]

## 竞品态势判定
[若有 competitor_matrix:分析竞品共性与差异化空间]
[若无:说明差异化判定基于推断,未经验证]
**差异化判定基础**: [competitor_matrix / inferred]

## 品牌战略机会点
### 情感化价值方向
[用户内在需求 + 产品价值交集]

### 定位候选方向
1. [方向A]:优势/劣势/适用场景
2. [方向B]:优势/劣势/适用场景
3. [方向C]:优势/劣势/适用场景

### 差异化空间
[基于竞品态势或产品特性的差异化机会]

### 取舍建议
[推荐方向 + 理由 + 放弃了什么]
```

## 约束

- 情感化价值必须指向用户内在感受,不能只是功能描述
- 竞品矩阵缺失时,必须明确标注差异化判定基于推断
- 定位候选方向至少 3 个,覆盖不同战略路径
- 取舍建议必须说明选择某方向放弃了什么
