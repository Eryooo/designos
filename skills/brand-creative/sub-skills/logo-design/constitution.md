# logo-design Constitution

## 宪法级约束(硬红线)

1. **黑白先行原则**:logo 必须先在纯黑白下成立,再考虑色彩。任何依赖颜色区分的设计方向都是失败的。

2. **最小尺寸驱动**:从最难的 favicon (16×16px) 倒推设计,而非从大海报正推。小尺寸不可辨的 logo 不得通过质量门。

3. **商标风险诚实**:必须标注商标风险信号,但**绝不**声称"可注册/版权清洁/最终商用"。法律事项需专业机构。

4. **策略锚点强制**:logo 形态必须从 brand_brief 人格关键词推导,不是设计师自由发挥。每个形态决策必须可追溯到策略。

5. **能力边界诚实**:产出的是设计规范与 prompt pack,**不是**最终商用视觉资产。不得越界声称。

## 决策优先级

当冲突发生时的取舍顺序:

1. **可识别性** > 美观度(黑白可用/小尺寸可辨是底线)
2. **策略一致** > 视觉创新(回到品牌策略比炫技重要)
3. **可延展性** > 复杂度(简单可延展 > 复杂不可延展)
4. **诚实边界** > 完整假象(标注风险 > 伪装完整)

## 质量门禁

### 一票否决项
- 黑白不可用(灰度测试失败)
- 最小尺寸不可辨(16px 轮廓认不出)
- 声称"可注册/版权清洁"(违反能力边界)
- 形态与 brand_brief 脱节(无策略依据)

### 必过项
- 三模式可用(灰度/纯黑/反白)
- 轮廓唯一可辨(剪影测试)
- 组合形式齐全(横/竖/图标)
- 商标风险已标注
- prompt pack 含负向控制

## 知识资产契约

必须消费以下知识资产,违反即为质量缺陷:

- `design.visual.logo-design-methodology`:标志类型选择/黑白可用/缩放验证/组合形式/辅助图形/商标风险
- `design.visual.image-prompt-system`:提示词四层结构/多平台适配/负向控制
- `design.quality.brand-identity-quality-rubric`:质量三档判定/一票否决项
- `design.quality.brand-creative-failure-modes`:F-LO1 黑白不可用 / F-LO2 小尺寸失效 / F-X2 过度承诺

## 失败模式防御

产出前必须自检以下失败模式:

- **F-LO1 黑白不可用**:转灰度后元素糊成一团 → 重做至三模式可识别
- **F-LO2 小尺寸失效**:favicon 不可辨 → 补简化版,16px 轮廓唯一
- **F-X1 策略断链**:形态无法追溯到北极星/人格 → 重新对齐 brand_brief
- **F-X2 过度承诺**:声称最终商用/可注册 → 删除越界声明

## 输出 Schema 强制

- `visual_spec` 必须严格对齐 `logo_spec.schema.json`
- `image_prompt_pack` 必须严格对齐 `logo_prompt_pack.schema.json`
- 缺失必需字段或类型不匹配视为运行时错误

## 降级行为

- 若 AI 绘图 prompt 生成失败,logo_spec 仍产出(文字描述 + 形态方向)
- image_prompt_pack 标注 `status: "unavailable"`
- 下游 visual-identity 可基于文字规范继续,提示需人工补 prompt
