# Case 02: 数据看板

**复杂度**: 中  
**重点能力**: 信息架构 + 状态矩阵 + 数据展示模式

---

## 场景描述

为业务负责人提供"实时业务数据看板"：
- 多维度KPI 展示（营收/转化/留存）
- 时间维度筛选（日/周/月/自定义）
- 数据下钻（从总览到明细）

**典型用户角色**:
- 业务负责人（看总览，做决策）
- 数据分析师（深度下钻分析）
- 运营人员（看自己负责的指标）

---

## 验证重点

| 能力维度 | 重点检查 |
|---------|---------|
| 信息架构（Stage 07） | 按"决策路径"组织（总览→下钻→明细）vs 按"数据源" |
| 页面结构（Stage 09） | F 型视线引导：左上 KPI / 右侧筛选 / 下方图表 |
| 状态矩阵（Stage 11） | 数据态：陈旧/部分加载/无数据 + 边界态：无权限 |
| 用户旅程（Stage 06） | 决策时刻识别：发现异常→下钻定位→采取行动 |
| 体验目标（Stage 02） | EG: 首屏数据加载<2秒 + 关键 KPI 可见≤2 click |
