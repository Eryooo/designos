# Case 03: 审批工作流

**复杂度**: 中高  
**重点能力**: 业务流程 + 异常处理 + 多角色协作 + 隐藏任务识别

---

## 场景描述

企业内部"通用审批工作流"系统：
- 员工提交申请（请假/采购/报销/合同）
- 多级审批（部门负责人→财务→HR→总经理）
- 审批结果通知 + 钉钉/飞书集成

**典型用户角色**:
- 申请人（提交/查看进度/补充材料）
- 审批人（审批/驳回/转交/退回修改）
- HR/管理员（查看全局/数据导出）

---

## 验证重点

| 能力维度 | 重点检查 |
|---------|---------|
| 业务流程（Stage 05） | 状态机: 草稿→提交→各级审核→通过/驳回/撤回；超时自动加签 |
| 用户任务（Stage 04） | hidden_tasks: 紧急加急/转交他人/打回修改 |
| 权限矩阵（Stage 05） | 三维: 状态×角色×操作（如"已驳回"状态下申请人能改但审批人不能） |
| 异常处理（Stage 05） | 审批人离职/超时/并发审批 |
| 用户旅程（Stage 06） | 情绪曲线：等待审批的焦虑 → 通过的释放 |
| 交互规则（Stage 12） | 危险操作（驳回/撤回）二次确认；多审批人并发处理 |
