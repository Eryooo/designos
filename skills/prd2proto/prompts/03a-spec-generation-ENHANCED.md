# Stage 03a增强: 设计规范生成（资深设计师级别）

## 🎯 资深设计师思维注入

在生成设计规范前，**先用视觉翻译思维审视需求**（方法论13）：

### 1. 视觉层级的资深判断

**Junior 陷阱**：所有元素一样大，没有层级  
**Senior 原则**：建立清晰的视觉层级（主标题/次标题/正文/辅助文字）

**执行前自检**：
- ✅ 定义至少 4 级字号（H1/H2/H3/Body）
- ✅ 主要操作按钮比次要操作明显
- ✅ 关键信息用颜色/大小突出

### 2. 色彩系统的资深判断（方法论31）

**Junior 陷阱**：选几个好看的颜色  
**Senior 原则**：建立系统（主色/辅助色/中性色/功能色）+ 对比度检查

**执行前自检**：
- ✅ 主色符合品牌（如有）
- ✅ 对比度符合 WCAG 2.1 AA 标准（≥4.5:1）
- ✅ 功能色明确（success/warning/error/info）
- ✅ 考虑色盲友好（不只靠颜色区分）

### 3. 组件状态的资深判断（方法论11）

**Junior 陷阱**：只设计正常状态  
**Senior 原则**：所有交互组件必须有 7 种状态

**强制要求**：
```
交互组件 7 种状态:
1. 默认 (default)
2. 悬停 (hover)
3. 按下 (active)
4. 聚焦 (focus)  ← Junior 最容易漏
5. 禁用 (disabled)
6. 加载 (loading)
7. 错误 (error)
```

## 💡 增强的输出要求

在原有 design-spec.md 基础上，增加：

### 颜色系统增强

```markdown
## 颜色系统（增强版）

### 主色（Primary）
- **值**: #1890FF
- **用途**: 主要按钮、链接、重要信息
- **对比度**: 
  - 白底: 3.3:1 (AA 通过)
  - 黑底: 8.6:1 (AAA 通过)
- **色盲测试**: ✅ 红绿色盲可区分

### 功能色
- **Success**: #52C41A（对比度 3.8:1）
- **Warning**: #FAAD14（对比度 2.9:1，需配合 icon）← 对比度不足，标注风险
- **Error**: #F5222D（对比度 4.2:1）
- **Info**: #1890FF（复用主色）

### Junior 易犯错误（已避免）
- ❌ 对比度不足（浅蓝底白字看不清）
- ❌ 只靠颜色区分（色盲用户无法辨别）
- ❌ 功能色不统一（有的地方 success 是绿色，有的是蓝色）
```

### 字体系统增强

```markdown
## 字体系统（增强版）

### 字号层级
| 层级 | 字号 | 行高 | 用途 | 对比度要求 |
|------|------|------|------|-----------|
| H1   | 24px | 32px | 页面主标题 | ≥7:1 (AAA) |
| H2   | 20px | 28px | 区块标题 | ≥4.5:1 (AA) |
| H3   | 16px | 24px | 小节标题 | ≥4.5:1 (AA) |
| Body | 14px | 22px | 正文 | ≥4.5:1 (AA) |
| Caption | 12px | 20px | 辅助文字 | ≥3:1 (可接受) |

### 中西文配对
- **中文**: PingFang SC / Microsoft YaHei
- **西文**: Helvetica Neue / Arial
- **等宽**: Consolas / Monaco（用于代码）

### Junior 易犯错误（已避免）
- ❌ 正文小于 12px（移动端难读）
- ❌ 行高过小（文字拥挤）
- ❌ 中英文字体不匹配（中文用 Arial 很丑）
```

### 组件状态规范增强

```markdown
## 组件状态规范（增强版）

### Button 状态完整示例

| 状态 | 视觉表现 | 交互行为 | 可访问性 |
|------|---------|---------|----------|
| 默认 | 蓝色背景，白色文字 | 可点击 | aria-disabled=false |
| 悬停 | 背景加深 10% | cursor: pointer | - |
| 按下 | 背景加深 20%，轻微缩放 | - | - |
| 聚焦 | 2px 蓝色外框 | 键盘可触发 | aria-focused=true |
| 禁用 | 灰色背景，灰色文字 | cursor: not-allowed，阻止点击 | aria-disabled=true |
| 加载 | 转圈 icon，禁用点击 | 显示"加载中..." | aria-busy=true |
| 错误 | N/A（按钮本身无错误态）| - | - |

### 状态转换逻辑
```
默认 → hover（鼠标移入）
hover → active（点击）
active → loading（提交中）
loading → 默认（成功）或 error（失败，由表单处理）
任何状态 → disabled（满足禁用条件）
```

### Junior 易犯错误（已避免）
- ❌ disabled 态未真正禁用（灰了但还能点）
- ❌ loading 态未禁用点击（重复提交）
- ❌ focus 态缺失（键盘用户无法导航）
- ❌ 状态转换突兀（无过渡动画）
```

## 🚨 强制检查清单（输出前自检）

### 色彩检查
- [ ] 所有文字对比度 ≥ 4.5:1（AA）
- [ ] 主要按钮对比度 ≥ 3:1
- [ ] 功能色明确且一致
- [ ] 标注了对比度不足的地方

### 字体检查
- [ ] 至少 4 级字号
- [ ] 正文 ≥ 14px（PC）或 ≥ 16px（移动）
- [ ] 行高合理（1.4-1.6）
- [ ] 中西文字体配对

### 组件状态检查
- [ ] 所有交互组件有 7 种状态
- [ ] disabled 态真正禁用
- [ ] loading 态禁止重复提交
- [ ] focus 态清晰可见
