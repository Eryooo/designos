# 截图描述（脱敏虚构版）

> 在真实场景下，`screens/` 目录会含 24 张实际截图。
> 测试时用文本描述代替（让评估器知道每张图的关键 UI 元素），
> 避免引入真实图像数据。

## 截图清单

### 工作台

#### 工作台-首页-默认.png
- 分辨率：1920x1080
- 关键元素：
  - 顶部导航：Logo / 工作台 / 数据治理 / 规则配置 / 设置
  - 左侧侧边栏：3 级菜单，「规则配置」位于第 3 级
  - 主区域：12 条混合待办（无视觉分类，字号颜色全相同）
  - 右上角：用户头像 + 通知图标
- 状态：default
- 痛点观察：12 条待办无视觉区分，难以快速定位规则相关待办

#### 工作台-首页-悬停.png
- 状态：鼠标悬停在第 3 条待办上
- 观察：仅有浅灰背景变化，无 cursor 变化、无 tooltip

### 规则草稿（M-001）

#### 规则编辑-默认.png
- 关键元素：
  - 表单：15 个字段（规则名、类型、字段映射、条件、生效范围 等）
  - 「保存草稿」按钮（次按钮，灰色边框）
  - 「保存并提交审批」按钮（主按钮，蓝色填充）
- 状态：default

#### 规则编辑-填写中.png
- 状态：用户已填 5 个字段
- 观察：填写时无 autosave 提示

#### 规则编辑-关闭后重开.png
- 状态：刷新页面后
- 观察：所有已填字段全部清空（草稿丢失）

#### 规则编辑-保存后.png
- 状态：点击保存后 2 秒
- 观察：按钮无 loading 状态、无 toast、无明显反馈
- 痛点：用户不知系统是否处理中

#### 规则测试-结果.png
- 关键元素：测试样本数据 100 条，规则生效条数 87 条
- 观察：但实际生效后是 92 条（与测试不一致）

### 版本管理（M-002）

#### 版本列表.png
- 关键元素：表格列出 8 个历史版本
- 观察：无版本对比按钮（v1.0 不支持）；当前版本无明显标识

#### 版本回滚-确认弹窗.png
- 关键元素：「确认回滚到 v3 吗？」+ 取消 / 确定
- 观察：无影响范围说明；无对比预览

### 审批流（M-003）

#### 审批列表-数据管理员视角.png
- 关键元素：10 条待审规则，按时间倒序
- 观察：无规则类型筛选；标题字段截断严重

#### 审批详情-规则查看.png
- 关键元素：规则字段以 JSON 格式展示
- 观察：非技术背景的审批人难以理解 JSON 字段

#### 审批通过-反馈.png
- 状态：点击「通过」后
- 观察：跳转回列表无明显成功提示
