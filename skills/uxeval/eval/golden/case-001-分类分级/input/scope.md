# 评估范围（scope.md）

## 1. 评估目标（一句话）

评估「数据规则配置平台 v1.0」对运营专员和数据管理员的体验质量，
找出影响「规则草稿 → 审批 → 上线」端到端流程的关键体验问题。

## 2. 重点关注的角色

- 运营专员（每天创建 / 调整规则，每周 5+ 次）
- 数据管理员（审批规则，每周 10+ 次）

## 3. 重点关注的模块

- 规则草稿（M-001）
- 版本管理（M-002）
- 审批流（M-003）

## 4. 不评估的部分（明确划界）

- 登录与权限管理（独立 SSO 系统，本期不评）
- v1.1 计划但未上线的功能（自动保存、协作锁定、规则市场、AI 推荐）
- 移动端（v1.0 不在范围）

## 5. 业务背景

v1.0 是规则配置能力的最小可用版本，重点验证主流程可用。
上一轮内测反馈集中在「规则保存丢失」「版本切换混乱」「审批流程不清晰」，
本期评估重点关注这三类痛点。

## 6. 已知问题

- 服务端响应慢（已立项后端优化）
- 邮件通知偶发延迟（运维已介入）
- 移动端适配（v1.1 范围）
