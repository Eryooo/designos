# 启发式原则库

> 本库按用户体验要素三层组织：表现层（视觉呈现）→ 框架层（界面设计）→ 结构层（导航与架构）。
> Stage 2（原则映射）时，AI 先分析产品特征，再从完整原则库中选取适用子集。

严重等级采用 4 级评分制：4级=-10 / 3级=-6 / 2级=-3 / 1级=-1 / 亮点=+1~+5。

---

## 表现层（视觉呈现）

### P1：美观而简洁的设计
**判定描述**：界面元素视觉清晰、层次分明、无冗余装饰。
**适用条件**：所有产品。
**典型违反**：
- 按钮、图标、文字过小或模糊
- 颜色对比度不足（WCAG AA 标准：正文 4.5:1，大字 3:1）
- 无意义装饰元素干扰主要内容
- 一屏出现多个等级别的主按钮，视觉重心混乱
**严重等级映射**：
- 4级（-10）：核心操作按钮不可见或无法点击
- 3级（-6）：重要信息难以识别，需要用户额外努力
- 2级（-3）：视觉噪音影响浏览效率
- 1级（-1）：美观度建议

### P2：不脱离现实（环境适配）
**判定描述**：界面符合用户使用环境的物理和认知约束。
**适用条件**：移动端、户外使用、特殊人群（老年/儿童/视障）。
**典型违反**：
- 移动端按钮小于 44×44pt（iOS）或 48×48dp（Android）
- 户外强光下低对比度配色
- 老年用户产品使用小于 14pt 字号
- 单手操作场景关键按钮放在屏幕顶部
**严重等级映射**：
- 4级（-10）：环境约束导致功能不可用
- 3级（-6）：需要切换环境才能完成任务
- 2级（-3）：环境适配不佳但可勉强使用
- 1级（-1）：环境适配建议

### P3：一致性与标准化
**判定描述**：同类元素在视觉、交互、文案上保持一致；遵循平台规范。
**适用条件**：所有产品。
**典型违反**：
- 同一操作在不同页面用不同图标/文案（"新建"、"创建"、"添加"混用）
- iOS 应用使用 Android Material Design 控件
- 主按钮在不同页面位置不一致
- 表格列宽 / 排序规则跨页面不一致
**严重等级映射**：
- 4级（-10）：不一致导致用户严重误操作
- 3级（-6）：不一致导致用户误操作
- 2级（-3）：不一致增加学习成本
- 1级（-1）：细节不一致

### P4：反馈与动效
**判定描述**：用户操作后立即获得视觉/听觉/触觉反馈；状态变化有过渡。
**适用条件**：所有产品。
**典型违反**：
- 按钮点击无视觉反馈（hover/active 状态）
- 长操作无进度提示
- 成功/失败无明确反馈
- 异步操作完成后没有通知
- 状态切换无过渡，体验生硬
**严重等级映射**：
- 4级（-10）：关键操作（支付、保存）无反馈，用户误以为系统卡死
- 3级（-6）：无反馈导致重复操作
- 2级（-3）：反馈不明确或延迟
- 1级（-1）：动效优化建议

## 框架层（界面设计）

### F1：预防出错
**判定描述**：通过设计减少用户犯错机会；高风险操作有二次确认；非法输入被前置拦截。
**适用条件**：所有产品，尤其是涉及数据删除、金额支付、权限变更的场景。
**典型违反**：
- 删除操作无二次确认
- 表单无输入校验（如邮箱格式、手机号）
- 破坏性操作按钮与常规按钮视觉无区分
- 不可点击的按钮未禁用，点击后才报错
**严重等级映射**：
- 4级（-10）：误操作导致数据丢失或资金损失
- 3级（-6）：误操作需要复杂恢复流程
- 2级（-3）：误操作可轻易撤销但仍打断流程
- 1级（-1）：预防机制建议

### F2：系统状态可见性
**判定描述**：用户随时知道"我在哪、我能做什么、系统在做什么"。
**适用条件**：所有产品。
**典型违反**：
- 多步流程无进度指示
- 当前页面在导航中无高亮
- 后台任务无状态提示
- 数据有筛选状态但没回显
**严重等级映射**：
- 4级（-10）：用户不知道任务是否完成，导致重复操作
- 3级（-6）：用户不知道当前位置，导航困难
- 2级（-3）：状态提示不清晰
- 1级（-1）：状态可见性建议

### F3：信息布局与可读性
**判定描述**：重要信息通过扫视即可获取；信息密度适中；视觉层级清晰。
**适用条件**：所有产品，尤其是数据密集型产品（BI、后台、表单）。
**典型违反**：
- 重要信息埋在次要位置
- 大段文字无分段/无标题
- 表格列过多无法扫视
- 所有字号/颜色相同，重点信息淹没
- 一次显示 20+ 个 KPI 指标，无主次
**严重等级映射**：
- 4级（-10）：核心信息完全无法识别
- 3级（-6）：关键信息难以找到，严重延迟任务
- 2级（-3）：信息布局不佳，增加认知负担
- 1级（-1）：可读性建议

### F4：帮助和说明
**判定描述**：复杂功能有上下文帮助；新手有引导；错误信息能帮用户理解原因并给出修复方向。
**适用条件**：B端产品、专业工具、低频高价值操作。
**典型违反**：
- 复杂表单无字段说明
- 专业术语无解释（用"实例 ID"替代"数据集编号"）
- 首次使用无引导
- 报错只有"系统错误，请联系管理员"，无错误码含义
- 帮助文档链接 404 或与实际功能不一致
**严重等级映射**：
- 4级（-10）：错误信息完全无意义，用户无法继续
- 3级（-6）：无帮助导致用户无法完成任务
- 2级（-3）：无帮助增加学习成本
- 1级（-1）：帮助不够清晰

### F5：对象关系可理解性（扩展）
**判定描述**：数据对象之间的层级、关联、依赖关系清晰可见。
**适用条件**：数据产品、配置平台、多层级系统。
**典型违反**：
- 父子关系不可见（如规则 → 规则组）
- 关联对象无链接跳转
- 依赖关系不可追溯
- 删除一个对象时不显示影响范围
**严重等级映射**：
- 4级（-10）：关系不清导致级联误操作（删一个连带删多个）
- 3级（-6）：关系不清导致误操作
- 2级（-3）：关系不清增加理解成本
- 1级（-1）：关系展示建议

### F6：前置条件表达（扩展）
**判定描述**：操作的前置条件明确告知；不满足时禁用并说明原因。
**适用条件**：工作流产品、审批系统、配置平台。
**典型违反**：
- 按钮可点但点击后报错"不满足条件"
- 前置条件不可见（如"需先配置 A 才能配置 B"）
- 禁用状态无 tooltip 说明禁用原因
**严重等级映射**：
- 3级（-6）：前置条件不清导致反复试错
- 2级（-3）：前置条件提示不明确
- 1级（-1）：前置条件表达建议

### F7：空状态引导（扩展）
**判定描述**：空列表/空结果有明确引导，告知用户"为什么空、怎么填充"。
**适用条件**：所有产品，尤其是新用户首次使用。
**典型违反**：
- 空列表只显示"暂无数据"
- 搜索无结果无建议（如检查拼写、放宽筛选）
- 新用户首次进入无引导
- 筛选后空结果未提示是筛选条件导致
**严重等级映射**：
- 3级（-6）：空状态无引导导致用户流失
- 2级（-3）：空状态引导不清晰
- 1级（-1）：空状态优化建议

### F8：高成本操作帮助（扩展）
**判定描述**：复杂、耗时、高风险操作有详细说明、示例和影响预告。
**适用条件**：B端产品、专业工具。
**典型违反**：
- 批量导入无模板下载
- 复杂配置无示例
- 高风险操作无影响说明（如"删除该规则将影响 X 个下游应用"）
- 长流程无中途保存草稿能力
**严重等级映射**：
- 4级（-10）：无帮助导致高成本操作完全失败
- 3级（-6）：无帮助导致操作反复试错
- 2级（-3）：无帮助增加操作成本
- 1级（-1）：操作辅助建议

## 结构层（导航与架构）

### S1：用户控制度与自由度
**判定描述**：用户可以撤销、返回、退出；不被系统强制锁定。
**适用条件**：所有产品。
**典型违反**：
- 无返回按钮
- 无撤销操作 / 无回收站
- 强制流程无法跳过
- 弹窗只有一个按钮无法关闭
- 长表单中途无法保存草稿
**严重等级映射**：
- 4级（-10）：用户被锁定无法退出，或不可逆破坏性操作无确认
- 3级（-6）：无撤销导致误操作不可恢复
- 2级（-3）：控制度不足影响效率
- 1级（-1）：自由度建议

### S2：再认而非记忆
**判定描述**：用户通过识别而非回忆完成任务；减少记忆负担。
**适用条件**：所有产品。
**典型违反**：
- 下拉框无默认值/历史记录
- 多步流程需要记住上一步输入
- 无最近使用/常用功能
- 选项需要用户记住格式（如"日期格式 YYYYMMDD"）
- 命令行式输入而无候选项
**严重等级映射**：
- 3级（-6）：记忆负担导致任务失败
- 2级（-3）：记忆负担增加操作成本
- 1级（-1）：再认体验建议

### S3：灵活性与效率
**判定描述**：为专家用户提供快捷方式；支持批量操作；高频功能可定制。
**适用条件**：高频使用产品、B端产品、专业工具。
**典型违反**：
- 无快捷键
- 批量操作只能一条条做
- 无搜索/筛选
- 无保存常用筛选条件 / 收藏夹
- 高频路径过深，每次需要多步导航
**严重等级映射**：
- 4级（-10）：核心高频任务每天耗费大量重复操作
- 3级（-6）：效率问题严重影响高频用户
- 2级（-3）：缺少快捷方式
- 1级（-1）：效率优化建议

### S4：多层级上下文（扩展）
**判定描述**：多层级导航时，用户随时知道"我在第几层、上层是什么、如何回到上层"。
**适用条件**：多层级产品（如文件管理、组织架构、分类系统）。
**典型违反**：
- 面包屑缺失
- 返回按钮不明确返回到哪一层
- 层级关系不可见
- 深层页面刷新后丢失上下文
**严重等级映射**：
- 3级（-6）：层级混乱导致迷失，需重新导航
- 2级（-3）：层级提示不清晰
- 1级（-1）：上下文展示建议

### S5：状态机可预测性（扩展）
**判定描述**：对象状态流转清晰可预测；用户知道"下一步会发生什么、能回退吗"。
**适用条件**：工作流产品、审批系统、订单系统。
**典型违反**：
- 状态流转无说明（如"草稿 → 提交 → 审核中 → 已发布"流程不可见）
- 操作后状态变化不可预测
- 状态回退无提示或不支持
- 状态变更无操作记录可追溯
**严重等级映射**：
- 4级（-10）：状态不可预测导致严重误操作（如误发布、误审批）
- 3级（-6）：状态不可预测导致误操作
- 2级（-3）：状态流转不清晰
- 1级（-1）：状态可预测性建议

## 使用方式

Stage 2（原则映射）prompt 要求 AI：

1. **分析产品特征**：
   - 类型：B端 / C端
   - 数据特征：数据密集 / 交互密集 / 内容浏览
   - 用户特征：新手为主 / 专家为主 / 混合
   - 终端：移动 / 桌面 / 多端
   - 关键场景：是否涉及高风险操作（支付、删除、权限）、是否多层级、是否有状态流转

2. **从完整原则库中选取适用子集**：
   - 优先选取"适用条件"明确匹配的原则
   - 表现层 P1-P4 一般全选（视觉基础项）
   - 框架层根据特征筛选（数据产品必选 F3/F5，工作流产品必选 F6/F8）
   - 结构层根据导航复杂度筛选（多层级必选 S4，工作流必选 S5）
   - 单次评估建议 8-12 条原则，过多会失焦

3. **输出选中的原则 ID 列表**（如 `[P1, P3, P4, F1, F2, F3, F5, S1, S2, S3]`）并把每条原则映射到 3-6 个最适用的模块。

4. **去重规则**：自定义原则（`inputs/principles.md`，编号 `HC-001+`）与内置原则语义重合时，保留自定义、删除内置（用户优先）。

## 示例

**产品**：数据规则配置平台（B端 / 数据密集 / 专家为主 / 桌面 / 多层级 + 状态流转）

**选取原则**：`[P1, P3, P4, F1, F2, F3, F4, F5, F6, F8, S1, S3, S4, S5]`

**Module 「数据规则配置」原则映射**（按优先级）：
1. F1 预防出错（规则错误会影响全量数据）
2. S1 用户控制度与自由度（必须支持版本回滚）
3. F2 系统状态可见性（生效需要异步执行）
4. F5 对象关系可理解性（规则 → 规则组 → 应用层级）
5. S5 状态机可预测性（草稿 / 生效 / 失效流转）
6. S3 灵活性与效率（高频用户需要规则模板）

## 注意事项

- 单次评估选取的原则不要超过 12 条，否则评估会失焦
- 同一问题尽量映射到 1 条主原则（避免"这个问题违反了 5 条原则"）
- 自定义原则不要重复造轮子，先看看 P/F/S 系列能不能覆盖
- 严重等级判断的核心：看用户能否完成任务、是否可恢复、影响范围多大
