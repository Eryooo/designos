# 人格建模法（Persona Modeling · M03）

> 通用资产 · `design.persona.persona-modeling` · status: pilot
> IP 设计第三阶段。把品牌人格从关键词升级为可观测、可一致执行的行为模式。
> MBTI 只作为辅助语言,不作为人格的唯一依据。

## purpose

为 IP 建立稳定可执行的人格:动机、恐惧、缺陷、成长弧、关系网络。让 IP 在不同场景下表现一致,而不是一套换皮的对话模板。

## input_contract

- 品牌北极星与人格关键词(来自 M01)。
- 世界观设定(来自 M02)。
- 性格倾向/语言风格偏好(可选)。

## decision_framework

1. **行为模式定位(主)**:从北极星与用户关系推导 IP 的核心行为模式——它如何识别需求、如何决策、如何交付。可用一个可观测的行为序列模型表达(如"识别 → 拆解 → 执行 → 提升"四步动机链)。
2. **MBTI 作为辅助语言(辅)**:用 E/I、S/N、T/F、J/P 四维度描述倾向,帮助团队对齐语言,但不得仅凭 MBTI 类型生成人格;允许融合型。MBTI 只回答"倾向",行为模式回答"具体怎么做"。
3. **动机与恐惧**:核心欲望、核心恐惧(可分层:能力/理解/价值/安全)、隐秘缺陷(让角色立体)。每个恐惧给应对方式。
4. **成长弧三阶段**:初始状态 → 关键事件(触发/过程/变化点) → 蜕变后,每阶段含能力/性格/关系变化。
5. **关系网络**:行动元六角色映射,明确 IP 与用户、系统、故障的关系。

## expert_heuristics

- 人格是行为模式不是标签;先定义"它会怎么做",再用 MBTI 贴标签。
- 只有动机没有恐惧/缺陷的 IP 是扁平的。
- 成长弧的关键事件必须有合理触发,不能突兀蜕变。
- 行为模式必须能反推到声音与视觉状态(与 `design.persona.voice-and-behavior-boundary`、M04 联动)。

## output_contract

- 人格档案(见 `design.templates.persona-profile`):行为模式 + MBTI 辅助 + 动机恐惧 + 成长弧 + 关系网。
- 行为模式序列(下游声音边界与视觉状态延展消费)。

## quality_rubric

| 分级 | 信号 |
|---|---|
| 高阶可评审 | 行为模式可观测且驱动声音/视觉,MBTI 仅辅助,动机恐惧成长弧自洽 |
| 中阶可用 | 人格要素齐全,但行为模式偏弱或过度依赖 MBTI |
| 低阶/不合格 | 仅有 MBTI 类型标签,无行为模式,无恐惧/缺陷/成长 |

## common_failure_modes

1. 仅凭 MBTI 类型生成人格,把标签当人格。
2. 只有正面动机,无恐惧/缺陷,角色扁平。
3. 成长弧蜕变突兀,无合理触发事件。
4. 行为模式无法反推到声音或视觉,人格停在文档。
5. 关系网络缺反对者/故障关系,IP 无张力。

## senior_review_checklist

- [ ] 存在可观测的行为模式序列,不只 MBTI 标签。
- [ ] MBTI 仅作辅助语言,允许融合型。
- [ ] 动机、恐惧(分层)、隐秘缺陷齐全。
- [ ] 成长弧三阶段且关键事件有合理触发。
- [ ] 行为模式可反推到声音/视觉。

## do_not_claim

- 不声明 MBTI 是科学人格测量;它只是辅助沟通语言。
- 不替代真实用户的心理画像研究。
- 不保证人格设定的市场共鸣。
