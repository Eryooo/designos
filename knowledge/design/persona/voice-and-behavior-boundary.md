# 声音与行为边界（Voice and Behavior Boundary · M03 配套）

> 通用资产 · `design.persona.voice-and-behavior-boundary` · status: pilot
> 把人格落成可一致执行的语言库与行为边界:说什么、怎么说、不能说、不能做。

## purpose

让 IP 的"声音"在所有场景下一致且可控。它把抽象人格转成可直接使用的口头禅、句式、情绪表达,并划定行为红线,避免 IP 在不同内容里人设崩塌。

## input_contract

- 人格行为模式与关键词(来自 `design.persona.persona-modeling`)。
- 世界观语境(来自 M02)。

## decision_framework

1. **口头禅库**:2–3 个标志性短语,与人格强绑定。
2. **场景句式库**:按场景分类(接令/执行中/完成/失败/待命/建议/提醒/协作),每场景给可复用句式。
3. **情绪表达**:若干情绪 × 语言表现(+ 可选视觉指示,如状态光效),与 M04 状态延展对齐。
4. **专业/语境分层**:技术、商务、协作等语境的语气差异。
5. **行为边界(红线)**:IP 绝不说什么(越界、承诺无法兑现、贬损用户)、绝不做什么(越权操作、不可逆动作不确认)。

## expert_heuristics

- 声音一致性比声音华丽更重要;一处崩人设比平淡更伤。
- 失败场景的话术最考验人格——是甩锅、道歉还是给下一步。
- 行为红线要具体到可判定,不能是"要友好"这类空话。
- 情绪表达应与视觉状态延展一一对应,避免语言与画面错位。

## output_contract

- 语言库(并入 `design.templates.persona-profile` 的语言部分)。
- 行为边界清单(下游内容 M05 与物料 M06 遵守)。

## quality_rubric

| 分级 | 信号 |
|---|---|
| 高阶可评审 | 覆盖 ≥6 场景,失败话术得体,行为红线可判定,语言与视觉状态对齐 |
| 中阶可用 | 句式齐全但情绪/红线偏笼统 |
| 低阶/不合格 | 只有口头禅,无场景分类,红线缺失或空泛 |

## common_failure_modes

1. 只有口头禅,无场景化句式,实际无法落地。
2. 失败场景话术甩锅或机械,人设崩塌。
3. 行为红线写成"要友好",无法判定。
4. 语言情绪与视觉状态不对应,画面与台词错位。
5. 不同语境(技术/商务)语气无差异,显得失真。

## senior_review_checklist

- [ ] 口头禅 2–3 个且与人格绑定。
- [ ] 场景句式覆盖 ≥6 类含失败场景。
- [ ] 情绪表达与 M04 状态延展对齐。
- [ ] 行为红线具体可判定。
- [ ] 语境分层语气有差异。
