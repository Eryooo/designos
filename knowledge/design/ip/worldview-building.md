# 世界观构建法（Worldview Building · M02）

> 通用资产 · `design.ip.worldview-building` · status: pilot
> IP 设计第二阶段。给 IP 一个自洽的世界,使其行为、能力、关系有来处。

## purpose

构建 IP 所处世界的时空、规则、关系与文化原型,让 IP 不是悬空的吉祥物,而是有边界、有动机来源的角色。世界观关键词向下约束视觉与内容。

## input_contract

- 品牌北极星与产品定位(来自 M01)。
- 文化原型偏好与时空意向(可选,缺失则按北极星推断)。

## decision_framework

1. **时空定位**:时代背景(古/今/未来/架空)、空间架构(物理/虚拟/混合)、3–5 个核心场所(氛围 + 功能)。
2. **世界规则三层**:能量规则(来源/消耗/上限)、能力规则(能做/不能做/如何成长)、社会规则(关系/位置/冲突来源)。
3. **角色关系网络**:用行动元模型(主体/客体/发送者/接收者/辅助者/反对者)画关系图。
4. **文化原型选择**:武侠/西游/赛博朋克/原创等 + 3–5 个核心意象词 + 转化逻辑(原型如何服务北极星)。
5. **世界观关键词**:3–5 个精炼词,供 M04 视觉转化消费。

## expert_heuristics

- 能力规则必须有边界,"万能 IP"等于没有性格。
- 世界规则要自洽:不能同时存在互相矛盾的设定。
- 文化原型是通用语言;具体世界设定(场所名、专有名词)是项目专属。
- 关系网络里"反对者/冲突来源"最容易被漏掉,但它定义了 IP 的张力。

## output_contract

- 世界观设定文档(见 `design.templates.worldview`),含时空/规则/关系图/原型/关键词。
- 世界观关键词清单(下游 M04 输入)。

## quality_rubric

| 分级 | 信号 |
|---|---|
| 高阶可评审 | 规则自洽有边界,关系网含冲突来源,原型转化逻辑清晰 |
| 中阶可用 | 设定齐全但能力边界模糊或缺冲突来源 |
| 低阶/不合格 | 世界规则矛盾,IP 万能化,原型与北极星无关 |

## common_failure_modes

1. 能力无边界,IP 万能,失去性格张力。
2. 世界规则互相矛盾,不自洽。
3. 关系网络缺反对者/冲突来源,世界没有张力。
4. 文化原型与目标用户认知不匹配(强行套用)。
5. 世界观关键词与后续视觉脱节,只停在文档里。

## senior_review_checklist

- [ ] 能力规则有明确"不能做什么"。
- [ ] 世界规则自洽无矛盾。
- [ ] 关系网络含冲突来源。
- [ ] 文化原型转化逻辑指向北极星。
- [ ] 世界观关键词可被 M04 消费。
