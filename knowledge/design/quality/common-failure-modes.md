# IP 设计常见失败模式（Common Failure Modes）

> 通用资产 · `design.quality.common-failure-modes` · status: pilot
> IP 设计 AI 产出最常见的跑偏与短板。每个失败模式给现象、识别信号、根因、防线。
> 这份清单是 AI 自评 + 资深评审都要对照的"反向 rubric"。

## purpose

把"AI 在哪里会塌房"显式列出。它替代旧库缺失的 failure case 集,让 AI 在产出前就能自检,资深设计师评审时也能快速定位短板。

## input_contract

- 任意阶段产出。
- 跨阶段一致性说明。

## decision_framework

每个失败模式按四段记录:**现象 / 识别信号 / 根因 / 防线**。下表为通用清单(分阶段编号),实际 AI 跑偏远不止这些,持续沉淀。

### 策略层(M01)
- **F-S1 北极星功能化**
  - 现象:北极星写成"帮用户写周报"。
  - 信号:北极星无情感/体验价值,只描述功能动作。
  - 根因:把 PRD 一句话当北极星。
  - 防线:M01 必过项卡五要素 + 情感价值,清单不过不放行。
- **F-S2 用户画像臆造**
  - 现象:列了画像但无真实痛点。
  - 信号:画像无具体数据/场景,推断未标。
  - 根因:无用户研究输入仍硬填。
  - 防线:推断显式标 `[inferred]`,缺失资料写 warnings。
- **F-S3 差异化靠贬低**
  - 现象:差异化变成"竞品做得不好"。
  - 信号:竞品分析含贬损语。
  - 根因:不研究竞品空白。
  - 防线:差异化必须基于空白而非贬低,清单卡。

### 世界观(M02)
- **F-W1 万能 IP**
  - 现象:能力规则没有边界。
  - 信号:无"不能做什么"。
  - 根因:为讨好怕给限制。
  - 防线:能力规则必过项含"不能做"。
- **F-W2 规则自相矛盾**
  - 现象:能量/能力规则前后冲突。
  - 信号:同一动作两种条件。
  - 根因:逐条写未做整体校对。
  - 防线:写完做规则一致性回扫。

### 人格(M03)
- **F-P1 仅 MBTI 标签**
  - 现象:只贴 ESTJ/INFP 等标签,无行为模式。
  - 信号:无可观测序列(识别→拆解→执行→提升 这类)。
  - 根因:把 MBTI 当唯一依据。
  - 防线:必过项强制行为模式;MBTI 仅辅助。
- **F-P2 无恐惧无缺陷**
  - 现象:只有正面动机,人格扁平。
  - 信号:无核心恐惧/隐秘缺陷字段。
  - 根因:怕"负面"破坏品牌。
  - 防线:必过项强制恐惧分层 + 隐秘缺陷。
- **F-P3 失败话术甩锅或机械**
  - 现象:执行失败时人设崩。
  - 信号:无失败场景台词或只说"系统错误"。
  - 根因:声音边界未覆盖失败场景。
  - 防线:声音边界必过项 ≥6 场景含失败。

### 视觉(M04)
- **F-V1 不继承品牌基因**
  - 现象:已有品牌做 IP 却另起炉灶。
  - 信号:无"继承/新增/禁冲突"三栏。
  - 根因:跳过基因盘点。
  - 防线:已有品牌时必过项卡三栏。
- **F-V2 32px 糊掉**
  - 现象:小尺寸下形象不可辨。
  - 信号:无尺寸分级简化或简化版无核心符号。
  - 根因:只设计标准尺寸。
  - 防线:必过项卡 32px 可辨 + 四级简化。
- **F-V3 风格漂移**
  - 现象:不同物料风格各异。
  - 信号:无风格谱系坐标。
  - 根因:每张图凭感觉发挥。
  - 防线:必过项卡风格坐标 + 锁定项。
- **F-V4 视觉跑偏(3D/二次元/动物化)**
  - 现象:AI 出图风格越界。
  - 信号:无严格禁忌或负向 prompt。
  - 根因:只写正向不写负向。
  - 防线:必过项卡禁忌 ≥5 + 负向 prompt 清单。
- **F-V5 色彩偏审美无依据**
  - 现象:色彩好看但与人格/世界观无关。
  - 信号:色彩无挂钩说明,且对比度未验。
  - 根因:把审美当依据。
  - 防线:色彩必挂钩 + 对比度验证。

### 提示词(M04 配套)
- **F-PR1 单段自由描述**
  - 现象:一句话当 prompt。
  - 信号:无四层结构。
  - 根因:不知有可控结构。
  - 防线:必过项卡四层。
- **F-PR2 模糊词导致不稳定**
  - 现象:每次出图差异大。
  - 信号:含"较亮/稍大/科技感"等模糊词。
  - 根因:未量化关键参数。
  - 防线:必过项卡量化与锁定项。
- **F-PR3 无负向控制**
  - 现象:频繁跑偏。
  - 信号:无 negative/严格避免清单。
  - 根因:只想"加什么"不想"去什么"。
  - 防线:必过项卡负向 prompt 清单。
- **F-PR4 跨图无基准**
  - 现象:三视图/状态图各画各。
  - 信号:无基准图或无一致性检查点。
  - 根因:没有锚点。
  - 防线:必过项卡基准图 + 检查点。

### 内容(M05)
- **F-C1 首秀平淡**
  - 现象:第一句台词与人格无关。
  - 信号:首秀目标未写或泛泛打招呼。
  - 根因:不当一回事。
  - 防线:必过项卡台词体现人格 + 目标明确。
- **F-C2 事件展功能不展关系**
  - 现象:核心事件只演示功能。
  - 信号:事件前后用户与 IP 关系无变化。
  - 根因:把功能当叙事。
  - 防线:必过项卡关系递进。

### 物料(M06)
- **F-M1 衍生品过度复杂**
  - 现象:工艺脱离量产能力。
  - 信号:清单无成本/可落地性。
  - 根因:只考虑识别度不考虑生产。
  - 防线:必过项卡可落地性与成本。
- **F-M2 印刷无 CMYK/Pantone**
  - 现象:实物严重偏色。
  - 信号:仅 RGB 标注。
  - 根因:不熟印刷流程。
  - 防线:必过项卡色彩模式。
- **F-M3 传播语调脱节**
  - 现象:不同场景人设不一致。
  - 信号:语调与声音边界冲突。
  - 根因:M03 与 M06 未联动。
  - 防线:必过项卡语调与声音边界一致。

### 跨阶段(总)
- **F-X1 关键词断链**
  - 现象:M01 关键词在 M04/M06 找不到落点。
  - 信号:跨阶段一致性说明缺失或对不上。
  - 根因:阶段独立推进未回看上游。
  - 防线:每阶段放行前做关键词回扫。
- **F-X2 假装完成不写 gap**
  - 现象:某阶段不达中阶仍标完成。
  - 信号:rubric 自评全高 + 无 gap report。
  - 根因:怕显得弱。
  - 防线:rubric 自评诚实 + gap 衔接。
- **F-X3 项目专属词污染通用层**
  - 现象:具体品牌词进入通用方法论。
  - 信号:`knowledge/` 通用文件含品牌专有名词。
  - 根因:照抄旧库未去专属化。
  - 防线:专属词只进 cases/,通用层结构测试拦截。

## expert_heuristics

- 失败模式不是稀有事故,是默认会发生;每个产出前都对照一遍。
- 模式比单点经验更稳定,持续把新跑偏沉淀回来比修单点更值。
- 防线必须可被自动或半自动检查,纯人工依赖会失守。

## output_contract

- 每个产出附一份失败模式自检清单(命中模式 + 处理方式)。

## quality_rubric

| 分级 | 信号 |
|---|---|
| 高阶 | 自检命中处全部处理或写入 gap,且持续沉淀新模式 |
| 中阶 | 自检覆盖全主类但根因/防线偏笼统 |
| 低阶 | 仅做事故复盘,不能防复发 |
| 不合格 | 不做自检 |

## common_failure_modes

(用于评估本资产的使用质量)
1. 把失败模式当读物,不做自检。
2. 命中失败模式后不处理也不 gap。
3. 不持续沉淀新模式,模式库陈旧。
4. 防线只写"要小心",无可操作动作。
5. 把所有问题归到"模型局限",不改流程。

## senior_review_checklist

- [ ] 每阶段产出附失败模式自检。
- [ ] 命中模式有去向(改 / gap)。
- [ ] 防线挂在必过项或测试上,不只是劝告。
- [ ] 新发现的跑偏已沉淀回模式库。

## examples_to_read

- 案例引用为可选项，且不得指向任何私有业务证据或项目专属 case 路径。
- 如需案例支撑，使用已批准的合成设计案例（synthetic case；后续在 `eval/golden-cases/` 下补充）。

## do_not_claim

- 不保证覆盖所有未来失败模式。
- 不替代真实跑偏样例库;每次新跑偏都应沉淀。
- 不保证防线零漏检。
