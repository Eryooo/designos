# 专业能力 Gap 报告（Professional Gap Report）

> 通用资产 · `design.quality.professional-gap-report` · status: pilot
> 这份报告是 IP 设计 AI 产出的"诚实声明书":
> **当前哪些阶段达不到中阶设计师水准、为什么、需要什么补充才能达到、临时放行的边界是什么。**
> 它存在的理由:不允许假装完成。

## purpose

让 AI 产出在某些维度不达中阶时,显式记录而非掩盖。它衔接 rubric 的低阶/不合格档与 stage-review-checklists 的未过项,把"已知短板"变成可被人类设计师/品牌方接管的清晰交接物。

## input_contract

- rubric 九维评分结果。
- 阶段必过项检查结果。
- 失败模式自检结果。

## decision_framework

任一以下条件成立 → 必须写 gap 报告,不允许标注完成:
1. rubric 任一维度低于"中阶可用"。
2. rubric 一票否决项(D2 差异化 / D6 法务 / D8 人格立体度)不合格。
3. 阶段必过项有未过且未即时返工。
4. 命中失败模式但未即时处理。
5. 关键输入缺失(无用户研究/无品牌资产/无竞品资料),依赖大量推断。

每条 gap 记录五段:
- **gap_id**:稳定编号,如 `GAP-V-001`(V 表视觉)。
- **stage / dimension**:发生于哪个阶段或哪个 rubric 维度。
- **现状**:当前产出的具体描述与缺位证据。
- **缺位原因**:输入缺失 / 资料不足 / AI 能力边界 / 流程未覆盖。
- **达到中阶所需补充**:具体动作(补用户研究 / 补品牌资产 / 增加评审 / 引入资深设计师 / 增加迭代轮次)。
- **临时放行边界(可选)**:在补足前可在哪些场景内有限使用,绝不可在哪些场景使用。

## expert_heuristics

- gap 不是失败,是诚实;掩盖才是真失败。
- 临时放行边界要保守:能不放行就不放行,涉及对外发布的资产更严。
- 每个 gap 都要有"被关闭"的明确路径与责任方。
- gap 报告与 rubric 自评、失败模式自检三者必须互相对应,不能互相打架。

## output_contract

- 一份按阶段/维度组织的 gap 列表。
- 总声明:**"本产出存在 N 个未关闭 gap,其中 M 个涉及一票否决项"** + 整体放行结论(可继续 / 需补充 / 不可放行)。

## quality_rubric

| 分级 | 信号 |
|---|---|
| 高阶 | 每个 gap 五段齐全且与 rubric/失败模式自检一致 |
| 中阶 | 列出主要 gap 但补充动作偏笼统 |
| 低阶 | 仅承认弱点无补充动作 |
| 不合格 | 应写 gap 而未写,或与 rubric 自评不一致 |

## common_failure_modes

1. rubric 自评全高,gap 报告为空,实际产出有明显短板(自欺)。
2. 列出 gap 但补充动作写"待优化",不可执行。
3. 临时放行边界宽松,涉及对外/印刷/法务仍放行。
4. 一票否决项不合格仍标整体放行。
5. gap 与失败模式自检对不上,自检命中却 gap 未列。

## senior_review_checklist

- [ ] rubric 任一低于中阶 / 一票否决不合格 / 必过项未过 → 已写入 gap。
- [ ] 每条 gap 五段齐全。
- [ ] 临时放行边界保守且具体。
- [ ] 整体放行结论与 gap 数量一致。
- [ ] 与 rubric 自评、失败模式自检三方一致。

## examples_to_read

- 案例引用为可选项，且不得指向任何私有业务证据或项目专属 case 路径。
- 如需案例支撑，使用已批准的合成设计案例（synthetic case；后续在 `eval/golden-cases/` 下补充）。

## do_not_claim

- 不替代品牌方/法务的最终风险评估。
- 不保证临时放行场景内不出问题。
- 不声明 gap 关闭即等于高阶质量,gap 关闭只是回到中阶起点。
