# IP 设计质量 Rubric（IP Design Quality Rubric）

> 通用资产 · `design.quality.ip-design-quality-rubric` · status: pilot
> 这是 IP 设计的多维评分标尺,用来回答一个最重要的问题:
> **这个 IP 方案是否合格,合格到哪一档,差在哪儿。**
> 它是判定"AI 产出能否进入真实项目讨论"的总闸门。

## purpose

把"好 IP / 不好 IP"从主观印象升级成可逐项打分、可定位短板、可决定是否放行的多维评估系统。它是资深设计师评审时实际会过的清单,旧库的核心短板就是没有这一层。

## input_contract

- 六阶段全部产出(M01–M06)。
- 跨阶段一致性说明(关键词从 M01 到 M06 的传导链)。
- 视觉资产与提示词产出。

## decision_framework

九大评分维度,每维四档(高阶可评审 / 中阶可用 / 低阶仅雏形 / 不合格)。

### D1 品牌一致性
- 高阶:北极星、人格词、世界观词、视觉关键词、传播语调环环相扣可逐项追溯。
- 中阶:主线一致但部分节点关键词漂移。
- 低阶:阶段间关键词各说各话,需大改。
- 不合格:无关键词链路。

### D2 差异化
- 高阶:差异化基于竞品空白,可一句话说清"为什么不是 X、不是 Y"。
- 中阶:有差异化但描述模糊。
- 低阶:差异化靠贬低或自夸。
- 不合格:无差异化或与竞品高度重叠。

### D3 可延展性
- 高阶:核心符号可在多状态/多尺寸/多物料中保持识别,延展边界明确。
- 中阶:延展可做但偶有跑偏。
- 低阶:每多一个延展就要重画。
- 不合格:无法延展。

### D4 识别度
- 高阶:32px 下可辨,远观凭轮廓认出,与竞品视觉不撞车。
- 中阶:标准尺寸可识别,小尺寸糊。
- 低阶:多尺寸均不易辨。
- 不合格:形象重复或与他者难以区分。

### D5 落地成本
- 高阶:衍生品工艺、印刷成本、产能可控,有首批清单。
- 中阶:成本可控但部分工艺过度复杂。
- 低阶:工艺要求脱离实际生产能力。
- 不合格:无法量产或成本失控。

### D6 法务/合规风险
- 高阶:核心符号无明显商标/版权风险信号,世界观无敏感映射,显式标注待法务确认项。
- 中阶:有潜在风险但已标注。
- 低阶:存在未标注的潜在风险。
- 不合格:明显抄袭/侵权/敏感映射。

### D7 提示词可控性
- 高阶:四层结构齐全,关键参数量化,负向项成清单,跨平台语义等价。
- 中阶:可控但参数部分模糊。
- 低阶:每次出图差异大。
- 不合格:无法稳定复现 IP。

### D8 人格立体度
- 高阶:行为模式 + 动机 + 恐惧/缺陷 + 成长弧 + 关系网齐全且自洽。
- 中阶:要素齐但偏扁平。
- 低阶:仅有 MBTI 标签或口头禅。
- 不合格:无可观测人格。

### D9 跨阶段证据
- 高阶:每阶段决策都有依据(用户研究/竞品/品牌资产/参考案例),推断处显式标注。
- 中阶:多数有依据,部分凭感觉。
- 低阶:大量凭感觉无依据。
- 不合格:无依据或编造依据。

## expert_heuristics

- 一票否决:D2 差异化、D6 法务、D8 人格立体度任一不合格,整方案不合格。
- 短板优先:总分高但某维度低阶,先补短板,不靠其他维度均分。
- 高阶 ≠ 完美:高阶意味着资深设计师"愿意在其上评审/微调",而非已是终稿。
- 自评要诚实:不达中阶的维度必须写入 `design.quality.professional-gap-report`。

## output_contract

- 一份九维评分表(每维档位 + 简短依据 + 短板说明)。
- 总判定:**高阶可评审 / 中阶可用 / 低阶仅雏形 / 不合格**。
- 短板项 → gap report 链接。

## quality_rubric

(本资产是 rubric 本身,以下用于评估 rubric 自身使用质量)

| 分级 | 信号 |
|---|---|
| 高阶 | 九维全打 + 依据 + 短板 + gap 衔接 |
| 中阶 | 九维全打但依据偏笼统 |
| 低阶 | 仅总评未分维 |
| 不合格 | 跳过自评 |

## common_failure_modes

1. 跳过自评直接说"已完成"。
2. 九维全打高分,无任何短板,缺乏自我批判。
3. 一票否决项不合格仍标"中阶可用"。
4. 自评后不衔接 gap report,短板没下文。
5. 用单维(如视觉)替代整体评分。
6. 把"齐全"等同于"合格",忽略证据与差异化。

## senior_review_checklist

- [ ] 九维全部打分且每维有简短依据。
- [ ] 一票否决项(D2/D6/D8)合格。
- [ ] 短板项已写入 gap report。
- [ ] 总判定与维度结果一致。
- [ ] 自评诚实,不掩盖弱项。

## examples_to_read

- 案例引用为可选项，且不得指向任何私有业务证据或项目专属 case 路径。
- 如需案例支撑，使用已批准的合成设计案例（synthetic case；后续在 `eval/golden-cases/` 下补充）。

## do_not_claim

- 不替代品牌方/法务终审。
- 不保证高阶评分等于市场成功。
- 不保证一次评分锁定终稿,迭代中可重打。
