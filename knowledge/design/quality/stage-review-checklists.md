# 阶段评审清单（Stage Review Checklists）

> 通用资产 · `design.quality.stage-review-checklists` · status: pilot
> 六阶段每阶的资深评审清单。每阶段产出落到这份清单上,
> 由资深设计师(或 AI 自评 + 人审)逐项过,**全过中阶,够中阶才放行下一阶**。

## purpose

让每个阶段都有可执行的"放行 / 不放行"判定,而不是凭感觉宣布完成。它把上层 rubric 的总评拆成阶段级的具体动作清单,避免短板被均分掩盖。

## input_contract

- 当前阶段的全部产出。
- 上游阶段的关键词清单(用于一致性检查)。

## decision_framework

阶段放行规则:**清单中所有"必过项"过 → 放行;任一"必过项"未过 → 必须返工或写入 gap;"加分项"未过不阻塞但记录。**

### M01 策略对齐 必过项
- [ ] 北极星含五要素(用户对象/方式/能力/价值/结果)且为情感/体验价值。
- [ ] 用户画像基于真实研究或推断已显式标注。
- [ ] 核心价值四层(产品/用户/组织/品牌)各 3–5 项。
- [ ] 人格关键词 3–5 个,每个有依据。
- [ ] 竞品 ≥3,差异化基于竞品空白而非贬低。

### M02 世界观 必过项
- [ ] 时空定位 + 3–5 个核心场所(氛围 + 功能)。
- [ ] 三层规则(能量/能力/社会)自洽且有边界。
- [ ] 关系网络含反对者/冲突来源。
- [ ] 文化原型转化逻辑指向北极星。
- [ ] 世界观关键词 3–5 个,可被 M04 消费。

### M03 人格 必过项
- [ ] 存在可观测行为模式序列,不仅 MBTI 标签。
- [ ] 动机 + 恐惧(分层)+ 隐秘缺陷齐全。
- [ ] 成长弧三阶段,关键事件触发合理。
- [ ] 行动元六角色映射齐全。
- [ ] 声音边界:口头禅 + ≥6 场景句式 + 失败话术 + 行为红线。

### M04 视觉 必过项
- [ ] 已有品牌:基因继承三栏(继承/新增/禁冲突)齐全。
- [ ] 主形/辅形有取舍依据,核心符号 1 个承担主要识别权重。
- [ ] 色彩 60-30-10 + 对比度达标 + 标注十六进制。
- [ ] 识别度:32px 可辨;小尺寸四级简化齐全。
- [ ] 状态延展 ≥4 状态,与情绪表达对齐。
- [ ] 风格谱系坐标明确(扁平/几何/极简/成熟商务等)。
- [ ] 严格禁忌项 ≥5 条且可判定。

### M04 提示词系统 必过项
- [ ] 四层结构齐全(核心符号/设计理念/技术参数/质量与避免)。
- [ ] 覆盖 ≥2 平台且语义等价。
- [ ] 关键参数量化(色值/百分比/像素/毫秒)。
- [ ] 基准图 + 跨图一致性检查点。
- [ ] 严格避免项成负向 prompt 清单。

### M05 内容 必过项
- [ ] 首秀台词体现人格且首秀目标明确。
- [ ] 核心事件 ≥3 且关系递进不重复。
- [ ] 对话语料覆盖 ≥6 场景含失败场景。
- [ ] 内容矩阵渠道匹配目标用户。
- [ ] 周期化内容节奏(可持续而非一次性)。

### M06 物料 必过项
- [ ] IP+logo 联合使用规范含最小尺寸 + 安全区。
- [ ] 禁止使用示例 ≥5,继承 M04 禁忌。
- [ ] 衍生品清单含可落地性与成本评估。
- [ ] 印刷物料标注 CMYK/RGB/Pantone。
- [ ] 传播语调与人格声音边界一致且分场景。

## expert_heuristics

- 必过项是底线,不是高阶要求;过不了就是不合格。
- 加分项不阻塞,但记录有助于后续打磨方向。
- 每阶段放行前对照上游关键词,断链就是必过项失败。
- 自检勾选必须给一句证据指向具体段落,不能空打勾。

## output_contract

- 每阶段一份评审记录:必过项打勾 + 证据指向 + 未过项的处理(返工 / gap)。

## quality_rubric

| 分级 | 信号 |
|---|---|
| 高阶 | 必过项全过且每项有证据指向 |
| 中阶 | 必过项全过但部分证据偏弱 |
| 低阶 | 部分必过项未过,已进入下一阶段 |
| 不合格 | 跳过清单或全凭感觉勾选 |

## common_failure_modes

1. 必过项勾打满但无证据,流于形式。
2. 未过项不返工不写 gap,直接进入下一阶段。
3. 上下游关键词断链未发现。
4. 把加分项当必过项,造成不必要返工。
5. 自检不诚实,把弱项算过。

## senior_review_checklist

- [ ] 每阶段必过项逐条评估。
- [ ] 每条勾选附证据指向。
- [ ] 未过项有去向(返工/gap)。
- [ ] 与上游关键词一致性核对。

## examples_to_read

- 案例引用为可选项，且不得指向任何私有业务证据或项目专属 case 路径。
- 如需案例支撑，使用已批准的合成设计案例（synthetic case；后续在 `eval/golden-cases/` 下补充）。

## do_not_claim

- 不替代品牌方与资深设计评审会的最终决议。
- 不保证清单全过即为终稿。
