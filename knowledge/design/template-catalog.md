# 设计模板目录（Template Catalog）

> 通用资产 · `design.template-catalog` · status: draft
> 登记可复用视觉风格模板族的**通用选择维度**。本文不内嵌任何 skill 的选择
> 算法,也不复制各模板的完整 token 正文——那些属于消费方 skill 的私有资产。

## 选择维度（通用）

挑选一个视觉风格模板时,可沿以下通用维度衡量(各维度互相独立,可组合):

- **产品形态**:后台/数据密集 vs 工具 vs 内容/文档 vs 消费品牌。
- **视觉气质**:专业克制 vs 活力年轻 vs 极简几何 vs 温暖可读。
- **信息密度**:高(大表单/表格) / 中(卡片化) / 低(大留白)。
- **明暗偏好**:优先暗色 / 仅浅色 / 两者皆可。
- **行业惯例**:稳重 vs 柔和 vs 热闹的领域默认期待。

## 仲裁原则（通用）

当维度冲突时,通用优先级:用户显式偏好 > 内容推断 > 行业惯例 > 中性默认。具体"哪个模板对应哪个分支"由消费方 skill 自己的选择算法决定,不在本目录内固定。

## 适用边界

本目录只描述"按什么维度选";模板的具体 token 体系、选择算法、微调白名单留在消费方 skill 内。
