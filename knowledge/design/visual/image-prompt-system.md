# AI 绘图提示词系统（Image Prompt System · M04 配套）

> 通用资产 · `design.visual.image-prompt-system` · status: pilot
> 把视觉规范转成可在多平台稳定、一致复现 IP 的提示词系统。
> 这是**可控性决策系统**,不是一句 prompt 模板。

## purpose

让 AI 绘图不是抽卡,而是可控复现:同一 IP 在不同平台、不同视图、不同状态下保持识别一致。它解决旧库提示词"缺多轮稳定性保证"的短板,把稳定性、一致性、严格避免项做成显式控制。

## input_contract

- 视觉规范(M04 `design.visual.visual-translation`):核心符号、色值、材质、风格坐标、状态延展、禁忌项。
- 目标平台清单(如 Midjourney / Stable Diffusion / 即梦 / 可灵 / Gemini 等)。
- 目标产出:三视图 / 状态图 / 物料图等。

## decision_framework

### 1. 提示词分层结构(平台无关骨架)
每条提示词按四层组织,顺序固定:
1. **核心符号定位**:主符号 + 视觉权重(如主符号占约 60–70%),高度符号化。
2. **设计理念表达**:风格坐标词(扁平/几何/极简/成熟商务等)、文化/世界观气质。
3. **技术参数**:色值(带百分比配比)、材质(如毛玻璃透明度)、光源方向、输出规格(分辨率/背景)。
4. **质量与避免**:必须达成项(32px 可辨、主色准确、风格统一)+ 严格避免项。

### 2. 多平台适配
- 同一四层骨架,按平台差异调整表达:
  - 自然语言型(即梦/可灵/Gemini/中文平台):用结构化中文描述 + 关键英文术语。
  - 参数型(Midjourney):核心描述 + 风格/比例/版本参数;权重用 `::` 等平台语法。
  - 可控型(Stable Diffusion):正向 prompt + **负向 prompt**(承载严格避免项)+ 必要的种子/控制说明。
- 保持跨平台**语义等价**:同一 IP 描述核心不变,只换平台语法。

### 3. 稳定性控制(单图可复现)
- **锁定项**:核心符号高保真、主色十六进制为绝对基准、材质参数量化(百分比/透明度)、动效幅度上限(如缩放 ≤30% 防变形)。
- 所有关键值给具体数字(百分比/毫秒/像素),不用"较亮""稍大"等模糊词。
- 提供种子/参考图策略(平台支持时固定种子或喂参考图)。

### 4. 一致性控制(跨图/跨视图/跨状态)
- 先定 1 张**基准图**(canonical),其余视图/状态以它为参照微调。
- 三视图、状态图共用同一核心符号定义与色值,只改姿态/光效/表情。
- 跨状态只允许在"状态延展"定义的范围内变化,不重画形象。
- 给"一致性检查点":符号是否同形、主色是否同值、风格坐标是否未漂移。

### 5. 严格避免项(负向控制)
- 把 M04 禁忌项转成每平台可用的负向表达(SD 负 prompt / MJ `--no` / 自然语言"严格避免")。
- 通用高频避免项:过度 3D、复杂机械结构、动物特征、二次元/萌系(若风格坐标为成熟商务)、魔法元素、多余装饰、文字水印、变形拉伸。

## expert_heuristics

- "必须做 + 严格避免"成对出现,比只写正向更能压住跑偏。
- 量化一切可量化的:模糊词是不稳定的根源。
- 一致性靠基准图 + 锁定项,不靠每次重描述。
- 平台会更新,骨架(四层 + 锁定项)比具体参数更值得沉淀。
- 负向 prompt 是 SD 类平台一致性的关键,不能省。

## output_contract

- 一套四层提示词模板 + 每个目标平台的适配版本。
- 基准图提示词 + 三视图/状态图衍生提示词。
- 锁定项清单 + 跨图一致性检查点 + 负向 prompt 清单。

## quality_rubric

| 维度 | 高阶可评审 | 中阶可用 | 低阶/不合格 |
|---|---|---|---|
| 结构 | 四层齐全且顺序稳定 | 有结构但层次混 | 一段自由描述 |
| 多平台 | ≥2 平台语义等价含负向 | 单平台可用 | 不分平台 |
| 稳定性 | 关键值全量化 + 锁定项 | 部分量化 | 全模糊词 |
| 一致性 | 基准图 + 检查点齐全 | 有基准无检查点 | 每图各画 |
| 避免项 | 负向项成清单可套用 | 有避免但零散 | 无避免项 |

## common_failure_modes

1. 一句自由描述当提示词,无分层,结果不可控。
2. 用"较亮/稍大/科技感"等模糊词,导致每次出图不一致。
3. 不分平台,把 MJ 参数塞进自然语言平台或反之。
4. 无基准图,三视图/状态图各画各的,识别不统一。
5. 无负向/严格避免项,频繁跑偏(3D/二次元/动物化/文字水印)。
6. 不锁定主色与核心符号,跨图色偏、形变。

## senior_review_checklist

- [ ] 四层结构齐全且顺序固定。
- [ ] 覆盖 ≥2 目标平台且语义等价。
- [ ] 关键参数全部量化,锁定项明确。
- [ ] 有基准图与跨图一致性检查点。
- [ ] 严格避免项成负向清单,可直接套用。
