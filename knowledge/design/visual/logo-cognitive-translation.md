# Logo 认知-感知转化链路(Logo Cognitive Translation)

> Domain: design | Type: methodology | Status: pilot
> DesignOS pilot synthesis(本资产为 pilot 阶段综合判断,非可追溯专业文献)。
>
> 本资产是 logo 设计的**认知翻译系统**,区别于 `design.visual.logo-design-methodology`
> (覆盖规范全流程)与 `design.visual.visual-translation`(偏 IP 形态映射)。它解决核心
> 短板:**如何防止"关键词直接跳符号"的文化联想陷阱**(如 growth→叶子、future→火箭),
> 转而通过**认知结构层**完成人类普遍感知的稳健翻译。

## purpose

让 logo 不是"关键词的视觉配图",而是**认知结构的图形化**:把抽象概念(如 growth /
future / coexistence)通过人类普遍的空间/关系认知模式(Verticality / Path /
Container)转化为图形关系,再操控视觉变量生成可识别、可评审的母形(mother shape)。
它解决的核心问题:**没有认知层的跳跃式翻译,会导致 logo 依赖文化符号(需解释才懂)、
堆砌元素(narrative overload)、或退化为装饰纹理(无图形机制)**。

## applies_to

- 品牌 logo 从关键词到图形的决策推导过程。
- logo 概念方案的资深评审(检查是否有完整认知链路,而非符号拼贴)。
- 产出 logo_spec 时,必须显式输出 cognitive_schema / graphic_relation /
  visual_variables / mother_shape 字段(中间决策痕迹)。
- 不适用:已有 logo 的规范化整理(无需重新推导认知链路)。

## input_contract

- 必需:品牌关键词(来自 brand_brief,如 growth / future / coexistence / trust)。
- 必需:目标受众的认知背景(B2B / C端 / 技术专家 / 大众市场)——决定感知层的解读方式。
- 可选:品牌差异化意图(避开哪些品类陷阱,如"不要看起来像教育 / 金融 / 医疗")。

## decision_framework

### 六步强制链路(不得跳步)

#### 1. keyword:原始抽象概念
- 品牌关键词(如 growth / future / coexistence / trust)。
- **禁止直接跳转**:不得在此步联想具体物体(如 seed / rocket / handshake)。

#### 2. audience_perception:目标受众感知
- 目标受众如何理解该概念,**不是字面义,而是心智感受**。
- 示例:
  - growth(成长) → B2B 受众感知为"持续向上的势能"(upward momentum)。
  - growth → C端年轻受众感知为"突破边界的扩张"(boundary expansion)。
- **判断标准**:写出受众的感知句,不含具体物体名词。

#### 3. cognitive_schema:底层认知结构
- 人类普遍的空间/关系认知模式,**不是视觉符号,而是认知框架**。
- 常用 schema(基于认知语言学):
  - **Verticality**(垂直性):上下、高低、升降(如成长、地位、进步)。
  - **Path / Journey**(路径/旅程):起点-路径-终点(如成长、未来、演进)。
  - **Container**(容器):内外、包含、边界(如保护、信任、社区)。
  - **Balance / Symmetry**(平衡/对称):稳定、对等、公平(如信任、协作)。
  - **Center-Periphery**(中心-边缘):核心辐射、集散(如生态、共存、网络)。
  - **Force Dynamics**(力动态):推拉、对抗、融合(如竞争、协作、共存)。
  - **Cycle**(循环):往复、迭代、可持续(如生态、创新、演进)。
- **为何强制此步**:schema 是防跳跃的关键层——它把关键词从文化符号领域(seed /
  rocket)拉入人类普遍空间认知领域(上升轴 / 路径 / 中心场)。没有 schema 层,设计师
  会直接从 growth 跳到叶子、从 future 跳到火箭——这是文化联想而非认知翻译。
- **判断标准**:选定的 schema 必须能用空间/关系词描述,不含具体形状名词。

#### 4. graphic_relation:图形关系
- schema 对应的图形关系,**尚未具体形状,仅结构关系**。
- 示例:
  - Verticality → lower-to-higher axis(下到上的轴)。
  - Path → origin-path-endpoint(起点-路径-终点三元素)。
  - Container → boundary-enclosure-opening(边界-封闭-开口)。
  - Center-Periphery → core-radiate-spread(核心-辐射-扩散)。
- **判断标准**:用结构词(axis / boundary / radiate)描述关系,不提形状(圆 / 箭头)。

#### 5. visual_variable:可操控的视觉变量
- 可操控的视觉变量,**符合战略意图,避开品类陷阱**。
- 常用变量:
  - **轴方向**(axis direction):垂直 / 水平 / 斜向 / 螺旋。
  - **开放性**(openness):封闭(完整边界) / 开放(缺口、延伸)。
  - **对称性**(symmetry):中心对称 / 轴对称 / 非对称(动态)。
  - **尺度级差**(scale hierarchy):单一尺度 / 渐变尺度 / 突变尺度。
  - **连接方式**(connection):独立元素 / 接触 / 交叠 / 融合 / 嵌套。
  - **负形比重**(negative space):实形主导 / 负形主导 / 实负等重。
  - **线性 vs 面性**(line vs area):线框 / 实心 / 线面结合。
- **战略意图绑定**:
  - 避开品类陷阱:金融常用"封闭+对称"(稳定),若要差异化则选"开放+非对称"。
  - 传达差异感知:科技创新→开放+非对称;传统信赖→封闭+对称。
- **判断标准**:每个变量的选择必须回答"为什么不选另一个"(如为什么开放不封闭)。

#### 6. mother_shape:候选母形
- 通过前述变量组合成**具体图形机制**,形成可识别的母形(mother shape)。
- **母形三检验**(必须全过,缺一否决):
  1. **意图感知**:目标受众在 3 秒内能否感知到预期意图(不依赖解释)。
  2. **品类差异**:是否避开了行业通用符号(金融盾牌 / 科技电路 / 教育书本)。
  3. **生产存活**:能否通过黑白测试(单色可识别)、小尺寸测试(16px 轮廓可辨)。
- **母形示例**(完整链路):
  - growth → upward momentum(感知) → Verticality(schema) → lower-to-higher
    axis(关系) → open upper endpoint + asymmetry(变量) → upward-opening arc
    with base anchor(母形,如"底部锚定的向上开口弧")。
  - coexistence → mutual shaping(感知) → Force Dynamics + Center-Periphery
    (schema) → overlapping forces with shared core(关系) → partial overlap +
    negative space balance(变量) → two interlocking shapes with visible
    negative gap(母形,如"双互锁形,可见负形间隙")。
- **判断标准**:母形必须可用简单几何描述,不依赖渐变/纹理/光效(generative
  artifact 不是图形机制)。

### 完整链路示例(强母形 vs 弱母形对比)

#### 示例 1:growth(成长)
**强母形(有完整链路)**:
1. keyword: growth
2. audience_perception: upward momentum(持续向上势能)
3. cognitive_schema: Verticality(垂直性)
4. graphic_relation: lower-to-higher axis with open upper endpoint
5. visual_variables: open upper endpoint(非封闭) + asymmetry(动态) + negative
   space at base(底部稳定,上部开放)
6. mother_shape: upward-opening arc anchored at base,轮廓为"底宽上开的弧形,顶部
   无封闭,侧面渐薄"
- first_impression: "向上生长""展开""未完成的上升"
- likely_misread: 若弧度过大可能读为"容器/杯";若底部不稳可能读为"漂浮"
- avoidance_rule: 不封闭顶部(会变成"完成态"),不对称底部(会失去稳定感)

**弱母形(跳步 / 符号跳跃)**:
- 直接跳转:growth → seed(种子) / leaf(叶子) / tree(树)
- 问题:literal transfer(字面转移),依赖文化符号(需解释"这是种子代表成长"),小
  尺寸下细节丢失(叶脉 / 种子纹理不可辨),无图形机制(只是物体简化)。

#### 示例 2:coexistence(共存)
**强母形(有完整链路)**:
1. keyword: coexistence
2. audience_perception: mutual shaping without losing identity(相互塑造但保持独立性)
3. cognitive_schema: Force Dynamics(力动态) + Center-Periphery(中心-边缘)
4. graphic_relation: overlapping forces with visible boundaries and shared
   negative core
5. visual_variables: partial overlap(部分交叠) + visible negative space
   (可见负形) + equal scale(等尺度,无主次)
6. mother_shape: two interlocking rounded shapes with central negative gap,轮廓
   为"两个圆润形部分互锁,中心留可见负形缝隙"
- first_impression: "相互影响""共享空间""各自完整但有连接"
- likely_misread: 若负形过小可能读为"融合为一";若交叠过浅可能读为"并列"
- avoidance_rule: 负形不可省略(否则无"共存"感,变"融合"),交叠深度需适中(过深
  失去独立性,过浅失去关系)

**弱母形(跳步 / 符号堆砌)**:
- 直接跳转:coexistence → four dots placed together(四个点并置)
- 问题:多概念并置,无结构融合(只是"多个"而非"共存"),无图形关系(点之间无互
  动机制),需解释"为什么是四个点"。

### 禁止跳跃模式(Forbidden Jump Patterns)

以下模式一律判定为**弱母形 / 不合格**:

#### 模式 1:keyword → familiar object(关键词直跳熟悉物体)
- 示例:growth → seed / leaf,future → rocket / star,education → book /
  graduation cap,trust → handshake / shield
- 问题:literal transfer(字面转移),只是物体相似,无认知翻译;依赖文化符号(需解
  释才懂);小尺寸下细节丢失(叶脉 / 火箭窗 / 书脊不可辨)。

#### 模式 2:keyword → symbol stacking(多概念符号堆砌)
- 示例:coexistence → four shapes placed symmetrically(四形对称放置),innovation
  + trust → lightbulb + shield(灯泡+盾牌)
- 问题:多概念并置,无结构融合;每个元素独立叙事,整体无母形;需逐一解释"这个代表
  X,那个代表 Y"(narrative overload)。

#### 模式 3:keyword → narrative elements(叙事元素堆砌)
- 示例:journey → path + mountain + sun(路径+山+太阳,每个都有故事),ecosystem
  → tree + river + animals(树+河+动物)
- 问题:narrative overload(解释多,感知弱);元素过多,缩小后不可辨;无单一母形
  (整体是"场景插画"而非"图形机制")。

#### 模式 4:skip to generative artifact(跳到生成装饰)
- 示例:依赖渐变 / 光效 / 纹理 / 毛玻璃 / AI 风格化,而无图形关系
- 问题:不过黑白测试(单色失效);不过小尺寸测试(装饰丢失后无轮廓);不过生产检验
  (印刷 / 刺绣 / 单色场景无法复现);无图形机制(只是"好看",无"可识别结构")。

## senior_heuristics

- **认知层不可跳**:没有 cognitive_schema 的方案,100% 会退化为文化符号拼贴。
- **母形先于美化**:先确定图形机制(轮廓+负形+关系),再谈装饰;反之是伪设计。
- **负形即语言**:母形的负形(gap / opening / enclosure)承载意图,不是填充物。
- **可描述即可评审**:若无法用结构词(axis / overlap / radiate)描述,则无法评审。
- **三检验硬卡**:意图感知 / 品类差异 / 生产存活,三者全过才是合格母形。
- **小尺寸倒逼**:从 16px 倒推图形机制,而非从大尺寸正推再简化(简化会丢机制)。

## output_contract

每个 logo 产物(logo_spec 或中间评审文档)必须显式输出以下字段(中间决策痕迹,供资深
设计师评审):

- `cognitive_schema`(string):选定的 schema 名称(如 Verticality / Path /
  Container),不得为空。
- `graphic_relation`(string):图形关系描述(如 lower-to-higher axis with open
  endpoint),用结构词不用形状词。
- `visual_variables`(object):操控的视觉变量及其取舍理由(如 `{"openness": "open
  upper endpoint", "rationale": "convey ongoing growth, avoid completion"}`)。
- `mother_shape`(string):母形的几何描述(如 upward-opening arc anchored at
  base),可用简单几何词。
- `first_impression_prediction`(string):预测目标受众 3 秒内的第一感知(不依赖解释)。
- `likely_misread`(array):可能的误读场景(如"若 X 则读为 Y"),用于后续调整。
- `avoidance_rule`(array):严格禁止事项(如"不封闭顶部""不省略负形"),直接进
  prompt 负向控制。

缺任一字段,或字段为"无 / 待定 / TBD",则不符合"资深可评审"标准,需补充完整认知链路。

## quality_rubric

| 维度 | 高阶可评审 | 中阶可用 | 低阶/不合格 |
|---|---|---|---|
| 认知链路完整性 | 六步显式输出,无跳步 | 有 schema 但跳 variable | 无 schema 或直跳母形 |
| schema 层质量 | 用空间/关系词,无形状词 | 有 schema 但混入符号 | 无 schema / 用物体词 |
| visual_variables | 每变量有取舍理由 | 列出变量但无理由 | 无变量 / 直接给形状 |
| 母形机制 | 几何可描述+三检验全过 | 有母形但过不了小尺寸 | 符号拼贴/装饰依赖 |
| 禁跳模式检查 | 无四类跳跃模式 | 轻微跳跃(如 variable 弱) | 明显跳跃(keyword→物体) |
| 可评审性 | 字段齐全可逐步推敲 | 字段不全但主链路在 | 无中间痕迹,只有结果 |

**一票否决**:无 cognitive_schema 字段 / schema 用物体词(如"种子""火箭") /
母形不过黑白或小尺寸测试 / 明显符号跳跃(如 growth→leaf)。

## common_failure_modes

1. **跳过 schema 层**:从 keyword 直接跳到 mother_shape(或跳到文化符号如 seed /
   rocket),无认知翻译。返工信号:问"为什么不是另一个符号",答不出结构理由。
2. **schema 用物体词**:cognitive_schema 字段写"树""火箭""握手",实为符号不是
   schema。返工信号:schema 里出现具体物体名词。
3. **visual_variables 空转**:列出变量但不解释取舍,或变量与 schema 脱节。返工信号:
   问"为什么选开放不封闭",答"好看"或"感觉对"。
4. **母形依赖装饰**:渐变/光效/纹理去掉后无轮廓,不过黑白测试。返工信号:转单色后
   不可识别。
5. **多概念堆砌**:试图在一个 logo 里表达 3+ 个关键词,每个词配一个元素。返工信号:
   需要逐一指认"这是 X,那是 Y"。
6. **负形缺失**:母形只有实形轮廓,无可识别负形(gap / opening / enclosure)。返工
   信号:填充纯色后与线框版无差异感知。
7. **误读未预判**:未输出 likely_misread 与 avoidance_rule,导致实际感知与意图偏离。
   返工信号:目标受众看完问"这是什么",与预期意图无关。

## senior_review_checklist

- [ ] 六步链路显式输出:keyword / audience_perception / cognitive_schema /
      graphic_relation / visual_variables / mother_shape?
- [ ] cognitive_schema 用空间/关系词(Verticality / Path / Container 等),无物体词?
- [ ] graphic_relation 描述结构关系(axis / overlap / radiate),无形状词(圆/箭头)?
- [ ] visual_variables 每项有取舍理由(为什么选 A 不选 B)?
- [ ] mother_shape 可用简单几何描述,不依赖渐变/纹理/光效?
- [ ] 母形通过三检验:意图感知(3 秒可感知) / 品类差异(避开行业通用符号) /
      生产存活(黑白+16px 可辨)?
- [ ] 输出 first_impression_prediction / likely_misread / avoidance_rule?
- [ ] 无四类禁止跳跃模式(keyword→物体 / 符号堆砌 / 叙事元素 / 装饰依赖)?

## integration_with_other_assets

- **与 `design.visual.logo-design-methodology` 关系**:本资产负责"关键词→母形"的
  认知翻译链路(第 0–1 步);logo-design-methodology 负责后续"母形→规范→应用→
  商标风险"的全流程(第 2–6 步)。两者串联使用。
- **与 `design.visual.visual-translation` 关系**:visual-translation 偏 IP 形态
  (角色/吉祥物),侧重人格映射与状态延展;本资产偏 logo 图形,侧重认知结构与母形机制。
  若品牌同时需 IP 与 logo,两资产分别调用,共享 cognitive_schema 层。
- **下游消费**:产出的 cognitive_schema / mother_shape / avoidance_rule 进入
  `design.visual.image-prompt-system`,生成稳定可复现的 AI 绘图提示词;first_impression
  与 likely_misread 进入品牌测试与迭代反馈。

## do_not_claim

- **不声称认知链路可替代专业 logo 设计师的视觉判断**(本方法提供决策框架与可评审痕迹,
  最终图形精修与品牌定稿仍需专业设计师)。
- **不声称 cognitive_schema 覆盖所有人类认知模式**(本资产列出常用 schema,复杂
  概念可能需组合或扩展)。
- **不声称母形即最终 logo**(母形是图形机制,最终 logo 需经矢量化、比例调整、品牌
  方定稿)。
- 不声明产出 logo 已商标查重或版权清洁(认知翻译不保证商标可注册性)。
- 不替代目标受众的真实感知测试(first_impression_prediction 为预判,需实测验证)。
