# 代码质量规则（Code Quality Rules）

> 通用资产 · `frontend.code-quality-rules` · status: draft
> 生成代码的通用质量规则。每条规则可被静态检查或评审验证,与语言中立。
> 本文不复制任何 skill 的 constitution 正文,也不定义 lint 配置。

## 规则

- **不硬编码视觉值**:颜色、间距、字号等通过 token 引用,不在组件里写死字面量。验证:能否在组件中找到字面量视觉值。
- **状态完整**:交互组件覆盖常见状态(空/加载/错误/正常/禁用)。验证:是否存在未处理的关键状态。
- **可达性基线**:可交互元素有可感知的焦点与语义。验证:键盘可达、有语义标签。
- **无死代码**:不保留未被引用的分支与变量。验证:静态检查可发现未使用项。
- **错误可处理**:异步与失败路径有明确处理,不静默吞错。验证:失败分支是否给出反馈。

## 适用边界

这些是跨语言/跨框架成立的通用质量底线;某 skill 的 constitution 可以在此之上加更严的项,但不应与本底线冲突。具体 lint 规则集与配置由消费方 skill 决定。

## 占位说明

本文为 K0 架构基线的 draft 占位,后续批次补全每条规则的检查方式与反例。
