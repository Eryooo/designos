# 方法论库（Methodology Library）

> 通用资产 · `research.methodology-library` · status: draft
> SWOT / PEST / KANO / JTBD / AIPL 等方法论的通用定义与适用判定。每个方法论
> 给出适用前提与不适用边界,互不混淆。本文不规定选择算法,也不绑定任何 skill
> 的方法论选择 stage。

## 方法论清单

- **SWOT**:盘点优势/劣势/机会/威胁。适用:对单一对象做内外部综合判断。不适用:需要量化优先级时。
- **PEST**:从政治/经济/社会/技术看宏观环境。适用:宏观趋势研判。不适用:产品级细节决策。
- **KANO**:把需求分为基本/期望/兴奋型。适用:需求优先级排序。不适用:缺乏用户反馈数据时。
- **JTBD（Jobs To Be Done）**:从"用户雇产品完成什么任务"看动机。适用:挖掘真实动机、发现替代方案。不适用:只需表面功能清单时。
- **AIPL**:认知→兴趣→购买→忠诚的用户链路。适用:消费链路与营销转化分析。不适用:非交易型产品的体验评估。

## 选择原则（通用）

- 只选能被现有证据支撑的方法论,数据不足不硬选。
- 每个选用的方法论登记:名称 / 为什么用 / 适用于哪部分结论。
- 方法论是手段不是目的,不为用而用。

## 适用边界

本文给定义与适用判定;具体在某次分析里选哪几个、按什么算法选,由消费方 skill 决定。
