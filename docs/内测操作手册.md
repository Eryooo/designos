# DesignOS UXEval — 设计师内测指南

> 给团队内部设计师，**2 步上手**

---

## 一、安装（一次性，30 秒）

打开终端（Mac 按 `⌘ + 空格` 搜索 "终端"），粘贴这一行：

```bash
curl -fsSL https://raw.githubusercontent.com/<YOUR_ORG>/<YOUR_INTERNAL_REPO>/main/install.sh | bash
```

脚本会自动：
- 下载 DesignOS skills 到 `~/.designos/skills/`
- 检测你的 IDE（Claude Code / Cursor / Trae 等）并配置全局识别
- 创建 `designos` 辅助命令

**不用装 Python，不用配 API Key。** IDE 里你已经配过的 Claude / GPT 模型就是 DesignOS 用的模型。

> Trae / Codex 用户额外一步：进入项目目录后运行 `designos inject`（注入 AGENTS.md）。

---

## 二、做一次体验评估

### 第 1 步：进入你要评估的项目目录

在 IDE 里打开你要评估的产品项目（任何目录都行），准备好：
- PRD 文件（.md / .pdf / .docx）
- 产品截图（5+ 张关键页面，客户端模式）

### 第 2 步：对 AI 说一句话

在 IDE/CLI 对话框输入：

> **`/uxeval`**

AI 会自动按流程执行：

1. 读 PRD → 推断评估范围（让你审阅 `scope.md`）
2. 推导用户角色 → 场景 → 旅程图 → **暂停问你「旅程对吗？」**
3. 生成体验任务清单 → **暂停问你「任务清单对吗？」**
4. 截图分析 → 启发式检测 → 问题归因 → **暂停问你「严重等级对吗？」**
5. 输出评估报告

**三个暂停点**回复 `继续` / `修改 <说明>` / `补充` 即可。

## 三、产出物

每次跑完，在当前项目目录的 `outputs/` 下：

| 文件 | 内容 |
|---|---|
| `旅程地图.md` | 用户旅程图 + 阶段表 |
| `任务清单-完整版.md` | 完整体验任务（给资深设计师） |
| `任务清单-简洁版.md` | 简版（给中阶设计师顺序执行） |
| `问题清单.md` | Markdown 问题清单（含严重等级 + 证据 + 建议） |
| `evidence/` | 截图证据 |
| `<时间戳>.xlsx` | Excel 报告（给产品/研发分派） |

## 四、常见问题

### 1. AI 没识别 `/uxeval`？

- **Claude Code**：重启会话后生效（skill 已全局安装）。
- **Cursor**：确认 `~/.cursor/rules/designos.mdc` 存在。
- **Trae / Codex**：在项目目录运行 `designos inject`，确认 `AGENTS.md` 存在。

### 2. 跑到中间断了

对 AI 说：「继续刚才的评估」或 `/uxeval resume`。

### 3. 评估方向跑偏（变成功能测试）

在 Checkpoint 时回复「修改」，告诉 AI：
> 「目前的任务清单偏功能测试了，请按"用户能否完成 XX 任务"重写」

### 4. 截图没法提供（客户端登录不了）

写一份 `screens-description.md`，文字描述每个页面。AI 会把它当虚拟截图处理。

```markdown
### 工作台-首页
- 顶部导航：Logo / 工作台 / 数据治理 / 设置
- 左侧侧边栏：3 级菜单
- 主区域：12 条待办（无视觉分类）
- 痛点：信息过载，无法快速定位
```

### 5. 我没有 Claude 账号 / IDE 用的是 GPT，能跑吗？

可以。`/uxeval` 流程用的就是 IDE 自身配置的模型。
**注意**：12 阶段 prompt 都针对 Claude 调优过，GPT-4o / DeepSeek 跑出来质量可能下降 20-30%。
推荐用 Claude Opus / Sonnet。

### 6. PRD 是 PDF / Word，能直接读吗？

可以。AI 会调用 IDE 自带的文件读取能力解析。
极端情况（扫描版 PDF）建议先用工具转成文字。

## 五、批量评估 / CI 集成（可选）

无人值守场景（CI 流水线、批量跑多个项目）下，`designos` 命令是另一个触发入口——做的事情和 IDE 里说 `/uxeval` **完全一样**，但不需要人工 Checkpoint 交互（用 `--auto-confirm` 跳过）。

```bash
pip3 install designos
designos run uxeval --mode client --workspace ./my-project --auto-confirm
```

日常评估在 IDE 里说 `/uxeval` 就够了。CLI 只是为了让同一套流程也能脚本化跑。

2. **质量**：哪个 stage 的输出不够好？（贴原文）

5. **使用流畅度**：哪一步卡住了？

反馈直接发到团队群或 GitHub Issue。
