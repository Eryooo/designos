# 基于终极目标的质量评估

**评估标准**: 是否达到"资深设计师产出标准"  
**评估日期**: 2026-06-09  
**核心原则**: 不走捷径，按质量标准排序

---

## 🎯 终极目标定义

**让 AI 像资深设计师一样工作**，具体指标：

### 指标 1：设计推理的专业深度
- 每个推理资产体现"资深设计师的专业判断"
- 不是"形式正确"，而是"内容专业"

### 指标 2：质量约束的真实有效性
- 质量门真的能拦截低质量输出
- 不是"质量门存在"，而是"真实约束 LLM"

### 指标 3：可追溯的完整性
- 每个关键决策都能追溯到依据
- 能真实回答"为什么这样设计"

### 指标 4：产出的可用性
- 达到"中阶可用、高阶可评审"
- 资深设计师愿意基于此评审/微调

## 🔴 阻塞终极目标的核心问题

### 核心问题 1：缺少"专业判断"的 Prompts

**为什么阻塞终极目标**：
- 当前 prompts（02-17）只有结构，没有"专业判断逻辑"
- LLM 不知道"资深设计师会如何思考"
- 即使有结构，也是"junior 级别"的填空

**举例说明**：

❌ **当前框架版**（junior 级别）：
```markdown
# user-task-modeling

## 推理过程
Step 1: 识别用户角色
Step 2: 列出任务
```

✅ **应该有的专业版**（senior 级别）：
```markdown
# user-task-modeling

## 资深设计师的思考方式

### 1. 不要直接问"有哪些任务"
错误：列出"增删改查"
正确：思考"用户的真实目标是什么"

### 2. 区分"功能"和"任务"
- 功能：系统能做什么（如"导出报表"）
- 任务：用户要达成什么（如"向老板证明业绩"）

### 3. 任务优先级的专业判断
不是按"功能重要性"，而是按：
- 频率 × 价值 / 完成成本
- 高频高价值低成本 = P0
- 低频高价值高成本 = 需要权衡

### 4. 识别"隐藏任务"
PRD 不会写的，但资深设计师知道：
- 错误恢复任务（如"误删后找回"）
- 批量操作任务（如"批量更新状态"）
- 协作任务（如"转交给同事"）

### 5. 任务的"情绪维度"
资深设计师会考虑：
- 哪些任务让用户焦虑（如"删除重要数据"）
- 哪些任务需要成就感（如"完成销售目标"）
```

**影响终极目标的程度**: 🔴🔴🔴🔴🔴（致命）

**为什么致命**：
- 没有专业 prompts，LLM 永远是 junior 水平
- 这是"AI 像资深设计师"的核心
- 不解决这个，其他都是空中楼阁

---

### 核心问题 2：质量门未真实约束 LLM

**为什么阻塞终极目标**：
- 当前 LLM 集成为 mock
- 质量门只验证 mock 数据
- 不知道真实 LLM 是否遵守约束

**举例说明**：

当前测试：
```python
# Mock 数据（人工构造的"完美"数据）
mock_output = {
    "user_goals": [{"goal": "...", "inferred": True}]  # 人工标注 inferred
}
# 质量门验证 mock → 通过
```

真实情况未知：
```python
# 真实 LLM 输出（可能不标注 inferred）
real_output = llm.call(prompt)  # LLM 可能忘记标注 inferred
# 质量门能否拦截？未验证！
```

**影响终极目标的程度**: 🔴🔴🔴🔴（严重）

**为什么严重**：
- 终极目标要求"质量可控"
- 但我们不知道质量门是否真的有效
- 必须用真实 LLM 验证
