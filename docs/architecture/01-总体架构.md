## 2. 分层职责

| 层 | 职责 | 变更频率 | 谁维护 |
|---|---|---|---|
| **用户入口** | 触发 Skill、展示 Checkpoint、收集确认 | 极低 | 各 IDE 原生支持 |
| **Skill** | 领域方法论 + Prompt + 评估集 | 高 | 设计专家 |
| **Kernel** | 通用引擎逻辑 | 低 | 架构维护者 |
| **MCP Server** | 具体工具实现 | 中 | 按需开发 |
| **基础设施** | 外部服务 | 不由我们控制 | — |

## 4. 关键数据流

### 4.1 单 Skill 执行

```
用户输入 → Config Loader → Pipeline Engine
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
         Stage 1         Stage 2    ...   Stage N
         (LLM call)      (MCP call)       (LLM call)
              │               │               │
              ▼               ▼               ▼
         Schema 校验     Schema 校验     Schema 校验
              │               │               │
              └───────┬───────┘               │
                      ▼                       │
              Checkpoint? ──Y──▶ 暂停等确认   │
                   │N                         │
                   ▼                          │
              写 outputs/ ◀───────────────────┘
                   │
                   ▼
              写 run.yaml (manifest)
                   │
                   ▼
              记录 trace
```

### 4.2 Skill 间串联

```
runs/001-ai-analytics/run.yaml
    │ outputs[type=design_strategy]
    ▼
Kernel 检测上游 → 提示用户确认引用
    │
    ▼
runs/002-prd2proto/pipeline
    │ upstream_refs[competitive_context] = 001 的产物
    ▼
Stage 1 prompt 中注入竞品上下文
```

### 4.3 组织记忆沉淀

```
设计师完成项目 → designos propose
    │
    ▼
Sanitizer 脱敏检查
    │ 通过
    ▼
git push staging 分支 → 自动开 PR
    │
    ▼
审批人 review → merge → main
    │
    ▼
其他设计师 git pull → 下次 Skill 执行时自动读取
```
