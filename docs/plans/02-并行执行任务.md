## 1. 并行策略

M1 阶段拆为 **4 条并行线**，每条线由独立 Sub-agent 负责：

```
                    ┌─────────────────────────────────────┐
                    │         主线（协调 + 联调）          │
                    │  负责：架构决策、接口定义、联调、     │
                    │        E2E 测试、你的 review 对接    │
                    └──────────────┬──────────────────────┘
                                   │
              ┌────────────────────┼────────────────────┐
              │                    │                    │
    ┌─────────▼─────────┐ ┌───────▼───────┐ ┌─────────▼─────────┐
    │   线 A: Kernel    │ │ 线 B: MCP     │ │  线 C: Skill      │
    │   Pipeline 引擎   │ │ Servers       │ │  UXEval 内容      │
    │   Checkpoint      │ │ pdf-parser    │ │  SKILL.md         │
    │   Config          │ │ excel-builder │ │  pipeline.yaml    │
    │   Error           │ │ browser-cap   │ │  prompts/         │
    │   Memory          │ │               │ │  reference/       │
    │   CLI             │ │               │ │  constitution     │
    └───────────────────┘ └───────────────┘ └───────────────────┘
```

---

## 2. 依赖关系

```
线 A (Kernel) ──────────────────────────────────────▶ 联调
    │                                                  ▲
    │ 提供接口定义（Week 2 Day 1 先出 interface）       │
    ▼                                                  │
线 B (MCP) ─────────────────────────────────────────▶ 联调
                                                       ▲
线 C (Skill 内容) ─────────────────────────────────▶ 联调
```

**关键依赖**：
- 线 B 和线 C **不依赖线 A 的实现**，只依赖线 A 的**接口定义**
- 因此线 A 第一天先输出接口文件（Pydantic Schema + 抽象类），线 B/C 即可并行开发
- 联调在 W5 进行，此时三条线合并

### 3.3 接口优先（Day 1 产出）

```python
# kernel/contracts/interfaces.py — 第一天就要定义好
from abc import ABC, abstractmethod
from pydantic import BaseModel

class StageResult(BaseModel):
    stage_id: str
    outputs: dict[str, Any]
    status: Literal["completed", "failed", "skipped"]

class PipelineContext(BaseModel):
    workspace: Path
    skill_name: str
    skill_version: str
    config: DesignOSConfig
    upstream_data: dict[str, Any] | None = None

class ILLMClient(ABC):
    @abstractmethod
    async def call(self, prompt: str, ...) -> str: ...

class IMCPClient(ABC):
    @abstractmethod
    async def call_tool(self, server: str, tool: str, args: dict) -> Any: ...

class IMemoryAdapter(ABC):
    @abstractmethod
    def read_session(self, key: str) -> Any: ...
    @abstractmethod
    def write_session(self, key: str, value: Any) -> None: ...
    @abstractmethod
    def read_organization(self, query: str, k: int = 5) -> list[Any]: ...
    @abstractmethod
    def propose_to_organization(self, category: str, payload: Any) -> str: ...
```

## 7. Sub-agent 开发规约

每个 Sub-agent 在开发时必须遵守：

```markdown
## 铁律（写进 Sub-agent system prompt）

1. 先写接口（Pydantic Model + ABC），再写实现
2. 先写测试，再写代码
3. pyright strict 全过
4. ruff check 全过
5. structlog 做日志，禁止 print
6. 异常继承 DesignOSError，带 ErrorCode
7. 配置走环境变量 + YAML，禁止硬编码
8. 单个函数 < 30 行，单个文件 < 300 行
9. 公共 API 有 docstring（Google style，1-2 行）
10. 每个模块有 README.md（接口 + 使用示例）
```

## 9. 风险预案

| 风险 | 触发条件 | 预案 |
|---|---|---|
| 线 A 延期 | Pipeline 引擎复杂度超预期 | 先实现最简版（顺序执行 + 无恢复），恢复逻辑 W5 补 |
| 线 B browser-capture 难做 | Playwright 环境问题 | 降级为「用户手动截图放到 inputs/screens/」 |
| 线 C Prompt 质量不够 | LLM 输出偏功能测试 | 先发 beta，M2 持续优化 Prompt |
| 联调接口不匹配 | Schema 定义有歧义 | Day 1 接口定义要足够详细 + 含示例 |
