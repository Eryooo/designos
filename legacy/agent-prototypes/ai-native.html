AI-native Operating Model

AI-native
产研交付机制

从“人肉交接”升级为“Issue 驱动、PR 承载、Agent 共创、人类决策放行”的连续交付系统。

AI 提效后的瓶颈，不在生成速度，而在共识、集成、质量与放行。
把体验探索、工程治理、质量安全、Agent 生产协同和灰度验证放进同一个可度量闭环。
更快验证价值 先做可体验版本，让需求从文档进入真实交互。
更稳工程落地 用契约、架构、可观测性把 Demo 收敛为系统。
更强质量治理 测试、安全、性能、可访问性前置并自动化执行。
更低发布风险 Feature Flag 灰度验证，指标达标再全量。
01

端到端交付主线

每个阶段都有明确产物、责任 owner 和自动化验证证据，避免需求、代码、测试和发布脱节。

1

立项 Issue

目标、用户故事、验收标准、成功指标
由 Agent 协助拆解任务、补全场景与风险。

2

体验探索

ux/<ticket>
甲方与 Agent 共创原型、文案、交互变体和 Mock。

3

宣讲 PR

Draft PR 做共识
Agent 辅助生成评审材料，乙方评估实现，丙方前置门槛。

4

契约锁定

API、数据、权限、错误码、埋点
Agent 辅助生成契约草案和一致性校验。

5

工程收敛

feature/<ticket>
乙方与 Agent 共创代码、测试、异常、日志、监控。

6

质量门禁

自动测试 + 安全 + 性能
丙方与 Agent 共创用例、规则、报告和阻塞 Issue。

7

灰度放量

Feature Flag
试用指标达标后进入 main 和全量发布。

02

AI Agent 生产协同与反馈闭环

Agent 进入甲乙丙生产现场参与产物生成，同时用 Issue/PR 闭环完成检查、分派、验证和愈合。

甲方生产增强

辅助生成需求卡、用户故事、验收标准、交互原型、文案变体、Mock 数据和埋点草案。

乙方生产增强

辅助生成接口契约、业务代码、状态模型、单测、异常处理、日志监控和重构建议。

丙方生产增强

辅助生成测试策略、E2E 用例、安全规则、性能预算、可访问性检查和放行报告。

反馈愈合闭环 让 AI 产出速度与质量反馈密度同步提升。

Issue 是最小管理单元

体验偏差、契约风险、工程问题、测试失败、安全漏洞和上线风险进入 Issue 池，自动打标、归因和分派。

PR 是解决证据载体

每个 PR 关联 Issue，说明变更、验证方式、风险、回滚和契约影响。Agent 预审后进入人工评审。

人类负责关键取舍

业务价值、架构方向、质量放行和灰度节奏由 owner 决策。Agent 提高效率，但不替代责任。

03

角色边界与两两闭环

甲乙丙不再串行交接，而是在同一功能分支和 Issue 系统里持续协同。

甲方

需求与体验 Owner

  • 业务目标与用户场景
  • 交互、视觉、动效
  • Mock 可演示实现
  • 验收标准与埋点需求
乙方

工程与系统 Owner

  • 接口契约与数据模型
  • 业务逻辑与权限边界
  • 异常处理、日志、监控
  • 性能、可维护性与 ADR
丙方

质量与安全 Owner

  • 测试策略与上线门槛
  • 回归、安全、性能检查
  • 可访问性与埋点验证
  • 放行、拦截与灰度观察
Agent

生产协同引擎

  • 参与甲方体验生产
  • 参与乙方工程生产
  • 参与丙方质量生产
  • 持续检查、反馈与愈合

体验完成

可演示、主流程闭环、关键状态覆盖、视觉交互对齐、文案基本稳定、埋点需求明确。

工程完成

接口联通、Mock 替换或隔离、权限完整、异常处理完整、日志监控可用、测试通过。

上线完成

质量门禁通过、Feature Flag 可控、灰度指标达标、高优风险关闭、回滚机制验证。

05

质量门禁与经营看板

把质量从“最后测试”升级为“可审计、可阻断、可度量”的经营能力。

T
自动测试 单测、集成、E2E、回归
S
安全扫描 SAST、依赖漏洞、权限边界
P
性能预算 延迟、加载、资源、容量
A
可访问性 键盘、语义、对比度、读屏
O
可观测性 日志、指标、告警、埋点
R
回滚就绪 开关、脚本、负责人、预案

集团可跟踪的关键指标

Lead Time立项到放行周期
PR 等待首次响应与合并时长
PR 规模变更行数与拆分质量
缺陷逃逸生产缺陷与回归遗漏
灰度回滚试用失败与回滚比例
Agent 贡献率生产协同、有效发现、自动修复

治理规则

1
所有需求必须 Issue 化没有目标、验收标准和成功指标，不进入实现。
2
所有上线必须可灰度、可回滚Feature Flag、负责人和退出条件是发布前置条件。
3
质量门禁有阻断权P0/P1 风险未关闭，不允许正式放量。
4
Agent 参与生产但 owner 负责AI 参与甲乙丙生产与反馈，人类承担业务、架构和质量决策。
最终目标：让 AI 带来的产出速度，真正转化为可控、可审计、可持续的集团级交付能力。 机制关键词：体验先行、契约先稳、工程收敛、质量放行、灰度验证、Agent 共创与愈合。
