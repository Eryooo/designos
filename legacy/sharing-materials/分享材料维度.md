**目的**：展示你如何突破传统体验设计的边界，主动拥抱业务上游（品牌策略、IP打造）和技术下游（Agent编排）。

### 二、问题拆解与解决思路（对齐“工作流程与范式变革能力”）

**目的**：展示你如何系统性地分析旧范式的问题，并设计全新的工作流范式。

**建议填充内容**：
- 旧流程痛点分析（可用表格或流程图）：
  | 阶段 | 痛点 | 根本原因 |
  |------|------|----------|
  | 策略对齐 | 需求模糊，反复沟通 | 缺乏结构化引导 |
  | 人格设定 | MBTI凭感觉定，不一致 | 无心理学工具支撑 |
  | 视觉转化 | 设计师主观性强 | 缺少人格→视觉映射规则 |
  | 品牌落地 | 物料各做各的 | 无统一规范 |
- 新范式设计思路：
  - 核心理念：**“品牌定位驱动设计目标 → 设计目标指引流程走向 → 方法论支撑工具选择”**。
  - 将专家经验方法论化、工具化、Agent化。
  - 设计六大Agent协同工作流（策略对齐、世界观构建、人格塑造、视觉转化、内容叙事、品牌创意落地），每个Agent有明确的输入、输出、确认点。
- 关键变革：从“线性、人工密集”到“结构化、人机协同、可复用”。

**可展示物**：新旧流程对比图、六大Agent协同流程图（Mermaid）。

### 三、实施路径与技术方案（对齐“AI工具应用能力”）

**目的**：展示你如何具体运用AI工具（大模型、MCP、知识库）实现Agent，以及解决过程中的技术难点。

**建议填充内容**：
- 技术选型：Trae AI IDE + Deepseek API + 自定义MCP工具。
- 核心组件：
  - 知识库：11个Markdown文档（6方法论+5工具包），每个文档含元数据、定义、步骤、模板、示例、注意事项。
  - Agent调度：Python脚本读取`index.json`，按依赖关系顺序调用方法论，调用多模态分析用户截图。
  - 人机协同：每步等待用户确认，支持中途补充信息。
- 关键实现：
  - 提示词工程：编写专业、结构化的Agent系统提示词（见前文）。
  - 多模态能力：用Deepseek视觉能力分析设计草图、竞品截图，提取布局、色彩、风格关键词。
  - 文件操作：自动读写`design-spec/ip-design/`下的阶段性产物。
- 难点与解决：
  - 如何保证方法论被准确执行？→ 拆分为独立Markdown文档，每次读取完整上下文。
  - 如何引导用户而不是一次性填空？→ 分步提问，每阶段只问少量关键问题。
  - 如何确保输出质量？→ 内置模板+示例+注意事项，强制AI对照。

**可展示物**：架构图、核心代码片段（可选）、知识库目录截图。
